Préférences - Langue


Les préférences de langue sont utilisées pour définir comment certaines fonctionnalités du langage pour la programmation seront affichées dans l'EDI de GameMaker Studio 2. Les options pour GML sont les suivantes:

  • Afficher les types d'arguments dans la barre d'état: Cette option concerne le moment où vous utilisez vos propres scripts personnalisés et avez utilisé la notation JSDoc pour identifier les différents arguments, etc... (voir ici pour plus d'informations). Lorsque cette option est cochée et que vous avez spécifié les types d'arguments dans le script, ils seront affichés dans la barre d'état en bas de l'éditeur, entourée de {}. Ceci est désactivé par défaut.

  • Afficher les descriptions des arguments dans la barre d'état: Si cette case est cochée, les descriptions que vous avez données à vos scripts personnalisés à l'aide de la notation JSDoc seront affichées dans la barre d'état en bas de la fenêtre de l'éditeur. Ceci est désactivé par défaut.

  • Afficher l'orthographe US pour les fonctions système: Par défaut, GameMaker Studio 2 utilise l'orthographe anglaise GB pour les fonctions, par exemple draw_text_colour, mais cela peut ne pas être exactement ce à quoi vous êtes habitué et ainsi vous pouvez passer aux orthographes américaines en utilisant cette option, de sorte que les fonctions deviennent draw_text_color, par exemple. Ceci est désactivé par défaut.
