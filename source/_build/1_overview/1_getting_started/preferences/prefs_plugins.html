Préférences - Plugins


Les préférences des plugins sont utilisées pour contrôler tous les plugins supplémentaires qui ont été ajoutés à l'IDE de GameMaker Studio 2. Par défaut, il y a une section dédiée au plugin Source Control pour l'intégration de projets GameMaker Studio 2 avec Git.

Cette section traite des préférences que vous devez définir pour que le plug-in Git SCM fonctionne avec GameMaker Studio 2. D'abord, vous avez l'ensemble des options générales:

  • Fenêtre de sortie du contrôle de source ouvert automatiquement: Lorsque cette case est cochée, la fenêtre de sortie du contrôle de source s'affiche automatiquement lorsque quelque chose change. Ceci est activé par défaut.

  • Verbose Logging: Activer la journalisation détaillée pour le journal de sortie Git. Ceci est désactivé par défaut.

Vient ensuite la section Identité de l' utilisateur avec les champs suivants à renseigner:
  • Nom d'utilisateur à valider en tant que: Vous devez entrer ici le nom d'utilisateur que vous utiliserez pour effectuer des validations avec Git (ce doit être le même que le nom d'utilisateur spécifié lors de la configuration de votre référentiel Git).

  • Email à commettre comme: L'email associé à cette identité Git.

Après avoir configuré l'identité de l'utilisateur, vous pouvez ajouter une nouvelle combinaison nom d'utilisateur / mot de passe pour l'authentification en cliquant sur le bouton intitulé Ajouter une nouvelle authentification Utilisateur / Passe, qui ouvrira la fenêtre suivante avec différents champs à remplir:

Vous pouvez également ajouter une nouvelle authentification par paire de clés SSH en cliquant sur le bouton avec cette étiquette. Cela ouvrira la fenêtre suivante pour que vous remplissiez:

En dessous de ces boutons, vous pouvez configurer l' outil de fusion et l' outil Diff pour SCM avec les options suivantes:

  • Outil Chemin d'accès à la fusion: spécifiez ici le chemin d'accès complet du fichier à l'outil de fusion Git.

  • Options de l'outil de fusion: Dans ce champ, vous pouvez ajouter des commandes supplémentaires à exécuter chaque fois que vous utilisez l'outil de fusion.

  • Chemin d'accès à l'outil Diff: vous spécifiez ici le chemin complet du fichier vers l'outil de comparaison Git.

  • Options de l'outil Diff: Dans ce champ, vous pouvez ajouter des commandes supplémentaires à exécuter lorsque vous utilisez l'outil Diff.

Finalement, après les avoir configurés, vous avez une dernière option:

  • Activer automatiquement le contrôle de la source: Lorsque cette option est cochée, le contrôle de la source sera automatiquement activé pour tous les nouveaux projets. Ceci est désactivé par défaut.
