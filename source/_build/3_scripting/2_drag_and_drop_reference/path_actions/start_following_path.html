Commencer le chemin suivant


La description

Cette action lancera une instance en suivant un chemin qui a été créé dans l'arborescence des ressources. Vous utiliseriez ensuite cette action pour indiquer à votre instance le chemin à suivre, la vitesse à suivre pour le suivre (mesurée en pixels par trame de jeu), comment se comporter quand elle atteint la fin du chemin, et si suivre l'absolu ou le relatif position du chemin. Lorsque vous définissez la vitesse, vous pouvez utiliser des valeurs négatives pour indiquer que l'instance doit suivre le chemin en sens inverse et noter que la vitesse définie ici peut être modifiée par le modificateur de vitesse défini dans l'éditeur de chemin pour chaque point de chemin. Définissez une vitesse de trajet sur 2 et dans le chemin, l'un des points a un modificateur de vitesse de 200%, les pixels réels par vitesse de trame de jeu à ce moment seront de 4.

Une fois que vous avez défini la vitesse du chemin initial, vous pouvez définir comment l'instance doit se comporter quand elle atteint la fin du chemin parmi les options suivantes (notez que lorsqu'une instance atteint la fin d'un chemin, cela déclenche également un autre chemin - Terminé Événement ):

  • Stop: termine le chemin, en arrêtant l'instance sur le dernier point de chemin.

  • Redémarrer: Continuer le chemin depuis le début, en sautant à la position de départ à nouveau si le chemin n'est pas marqué comme fermé dans l'éditeur de chemin.

  • Boucle: Continuez à partir de la position actuelle pour réexécuter le chemin.

  • Revers: Revenez sur le chemin à nouveau (en inversant la vitesse du chemin), et quand il atteindra le début du chemin, il s'inversera de nouveau, etc...

Enfin, vous pouvez marquer le chemin comme Relatif ou non. Par défaut, l'instance commencera à suivre le chemin exactement comme vous l'avez conçu et placé dans l'éditeur de chemin (la position absolue), mais si vous marquez la case Relative, elle commencera à suivre le chemin à partir duquel l'instance a été créé (la position relative). Arguments:

Argument La description
Path La ressource de chemin pour commencer à suivre
Speed La vitesse (en pixels par trame de jeu) à suivre le chemin
On End La réponse à atteindre la fin du chemin
