Définir les données de mosaïque au pixel


La description

Avec cette action, vous pouvez définir la mosaïque pour une cellule de tuile trouvée à une position spécifique sur une couche de tilemap. Le tiledata est simplement une valeur qui reflète l'index de la tuile ainsi que les valeurs de retournement, de miroir et de rotation, ainsi que tous les masques de tuiles personnalisés que vous avez utilisés. Avec cette action, vous devez d'abord fournir la couche à la cible en utilisant le nom de la couche (une chaîne) tel que défini dans l'éditeur de pièce, puis les axes x et y dans la pièce pour définir la mosaïque. Notez que le tiledata sera défini pour la "cellule" tilemap dans laquelle la position donnée atterrit. Par exemple, si vos tuiles de tileset sont 16x16, alors la couche tilemap contiendra des tuiles dans des cellules de 16x16 pixels, donc si vous utilisez cette action pour cibler la position de la pièce (8, 8), vous changerez la tuile dans la cellule 0, 0) de la tilemap.

REMARQUE: La couche sélectionnée doit avoir été définie comme une couche Tilemap dans l'éditeur de pièce, sinon vous risquez d'obtenir des erreurs.


Le dernier argument de l'action est le tiledata lui-même. Vous devez normalement récupérer le tiledata pour la cellule tilemap à l'aide de l'action Get Tile Data At Pixel, puis le manipuler à l'aide de l'action Set Tile Data Transform avant de le redéfinir avec cette action. Pour plus d'informations sur tiledata, veuillez consulter la section GML sur Tilemaps - Tile Functions.


Syntaxe de l'action:


Arguments:

Argument La description
Layer La couche avec le tilemap à cibler
X La position le long de l'axe horizontal (X) pour définir le tiledata pour
Y La position le long de l'axe vertical (Y) pour définir la mosaïque
Data Le tiledata à utiliser pour placer la cellule à la position
