Définir la variable d'instance


La description

Avec cette action, vous pouvez définir l'une des variables intégrées. Toutes les instances de tous les objets ont déjà un certain nombre de variables créées pour contenir des informations essentielles telles que la position actuelle x / y, la vitesse, la direction, la visibilité et bien d'autres choses. Les variables d'instance que vous pouvez utiliser sont:

  • Coordonnée X - La position le long de l'axe des x dans la pièce.

  • Coordonnée Y - La position le long de l'axe des y dans la pièce.

  • Direction - La direction du mouvement pour l'instance (la valeur par défaut est 0, où la droite est 0 °, la hauteur est 90 °, la gauche est 180 ° et la baisse est 270 °).

  • Vitesse - Vitesse (en pixels par image de jeu) sur laquelle l'instance se déplace.

  • Vitesse horizontale - La vitesse horizontale (en pixels par image de jeu) sur laquelle l'instance se déplace. Une vitesse négative signifie qu'elle se déplace vers la gauche, une vitesse positive est correcte et la valeur par défaut est 0.

  • Vitesse verticale - La vitesse verticale (en pixels par image de jeu) sur laquelle l'instance se déplace. Une vitesse négative signifie qu'elle remonte, une vitesse positive est en baisse et la valeur par défaut est 0.

  • Visible - La variable toggle de visibilité (par défaut est true sauf si défini dans l' éditeur d'objets, définissez sur false rendre invisible). Notez qu'une instance invisible exécute toujours tous ses événements à l' exception de l'événement Draw.

  • Persistant - La variable de bascule de persistance (la valeur par défaut est false sauf si défini dans l' éditeur d'objets, définissez sur true rendre persistant). Cela rendra une instance "persistante" d'une pièce à l'autre, c'est-à-dire que l'instance ne sera pas nettoyée à la fin de la pièce mais sera reportée dans la pièce suivante. Notez que lorsqu'une instance est persistante, elle n'appelle pas son événement Create et Destroy, mais appelle les événements Clean Up, Room End et Room Start.

  • Couche - Couche à laquelle l'instance a été assignée.

  • Profondeur - Valeur de profondeur affectée à la couche.

  • Sprite: ressource d'image-objet affectée à l'instance.

  • Image Index - L'index d'image de l'image-objet affecté. Cette valeur définit l'image de l'image-objet assignée à l'occurrence, et si la valeur est supérieure au nombre d'images de l'animation, elle «boucle» de sorte qu'une image-objet contenant 5 images si elle est définie sur l'image 7

  • Angle de l'image - L'angle sur lequel l'instance doit être définie (par défaut, 0, où la droite est 0 °, vers le haut 90 °, gauche 180 ° et vers le bas 270 °).

  • Image Alpha - La valeur alpha de l'instance (la valeur par défaut est 1) de 0 à 1, où 0 est complètement transparent et 1 est complètement opaque.

  • Image Blend - La couleur à fusionner avec l'image-objet de l'instance (la couleur par défaut est le blanc).

  • Vitesse de l'image - La vitesse d'animation pour l'instance.

  • Echelle horizontale - Echelle horizontale à laquelle l'image-objet de l'instance doit être dessinée (ceci affectera également la boîte de délimitation de l'instance et les collisions). Une échelle de 1 indique l' absence de mise à l' échelle (1: 1), les valeurs plus petites échelle vers le bas (0,5, par exemple, sera la moitié de la largeur de l'image - objet), les valeurs de plus grande taille plus grande échelle et les valeurs négatives vont refléter l'image - objet et l' échelle à moins que le La valeur utilisée est exactement -1 (dans ce cas, l'image-objet est simplement mise en miroir le long de l'axe Y sans mise à l'échelle).

  • Échelle verticale - Échelle verticale à laquelle l'image-objet de l'instance doit être dessinée (cela affectera également la boîte de délimitation de l'instance et les collisions). Une échelle de 1 indique l' absence de mise à l' échelle (1: 1), les valeurs plus petites échelle vers le bas (0,5, par exemple, sera la moitié de la hauteur de l'image - objet), les valeurs de plus grande taille plus grande échelle et les valeurs négatives vont retourner l'image - objet et l' échelle à moins que le La valeur utilisée est exactement -1 (dans ce cas, l'image-objet est simplement retournée le long de l'axe Y sans mise à l'échelle).

  • Masque - L'index du masque pour l'instance. La valeur par défaut est -1 (ce qui signifie que l'instance utilisera le masque de collision pour l'index d'images-objets actuellement affecté), mais si elle est définie sur une autre ressource, le masque de collision sera utilisé à la place de l'image-objet. indice.

  • Friction - Friction pour utiliser chaque image du jeu pour ralentir l'instance. Cette valeur sera simplement déduite de la valeur de vitesse actuelle de chaque image de jeu (ou ajoutée si la valeur de la vitesse est négative).

  • Gravité - La gravité applique une force constante dans la direction de la gravité (voir ci-dessous) de l'instance, influençant sa vitesse (la valeur par défaut est 0). Notez que la gravité est une force cumulative et accélérera l'objet si vous choisissez de ne pas plafonner la vitesse finale.

  • Gravity Direction - Ceci peut être utilisé pour définir la direction du mouvement lorsque la gravité des instances est supérieure à 0 (la valeur par défaut est 270). Notez que les directions dans GameMaker Studio 2 sont calculées comme GameMaker Studio 2: 0 ° étant à droite, 90 ° étant en hausse, 180 ° étant à gauche et 270 ° étant en baisse.

  • Score - Définir le score variable.

  • Vies - Réglez le lives variable.

  • Santé - Définissez le health variable.

Une fois que vous avez sélectionné la variable intégrée à utiliser, vous définissez la valeur de cette variable. La valeur que vous utilisez dépend de la variable sélectionnée (voir la liste ci-dessus pour les valeurs par défaut et les plages) et vous pouvez marquer la valeur par rapport à la valeur actuelle de la variable. Vous pouvez également ajouter des variables supplémentaires à définir dans la même action en cliquant sur l'icône plus à côté de l'action, et en sélectionnant une autre variable d'instance et en lui donnant une valeur.
