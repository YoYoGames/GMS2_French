Changer l'instance d'objet


La description

Avec cette action, vous pouvez changer une instance en une autre instance. Vous devez spécifier l'objet que l'instance change dans une autre instance de, et l'action changera l'instance, en exécutant l'événement Destroy et l'événement de nettoyage de l'instance initiale et l'événement de création de la nouvelle instance. Notez que si vous ne spécifiez pas de direction ou de vitesse etc... alors la nouvelle instance "héritera" de ces valeurs. Par exemple, si l'instance initiale a un image_angle (rotation) de 180, se déplace à 2px par tique de jeu et a un image_speed (vitesse d'animation) de 0.5, l'instance dans laquelle elle se trouve conservera ces valeurs, sauf si elle est explicitement définie dans l'événement Create de la nouvelle instance.

Il est à noter que le fait de changer l'instance signifie que vous ne pouvez effectuer aucune autre action avec cette instance avant le prochain tick du jeu, en essayant en particulier d'accéder aux variables etc... car cela entraînera une erreur. Fondamentalement, vous changez l'instance mais elle n'est pas réellement disponible jusqu'à la fin de l'étape en cours, donc pour accéder à l'une des variables qu'elle contient directement (par exemple, appeler inst_changed.x ) ne fonctionnera pas.
