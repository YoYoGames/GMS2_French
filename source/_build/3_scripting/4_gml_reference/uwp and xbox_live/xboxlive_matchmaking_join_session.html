xboxlive_matchmaking_join_session


La description

Cette fonction remplira une carte DS avec les détails des utilisateurs de la session spécifiée, ou -1 en cas d'erreur. Vous pouvez obtenir le pointeur d'ID de session à partir de la mappe DS async_load renvoyée dans l'événement social asynchrone lorsque vous avez trouvé une session). La carte DS renvoyée aura les paires clé / valeur suivantes:

  • " num_results " - contient le nombre d'utilisateurs dans la session

  • " userId <index> " - contient l'identifiant de l'utilisateur (<index> est une valeur de 0 à "num_results" - 1)

  • " userIsOwner <index>" - contient 1 si l'utilisateur est l'hôte, sinon 0 (<index> est une valeur de 0 à "num_results" - 1)

Lorsque la carte n’est plus nécessaire, elle doit être supprimée de la mémoire à l’aide du bouton ds_map_destroy() sinon, vous obtiendrez une fuite de mémoire.

Cette fonction est uniquement valide lors de l'exportation à l'aide de la cible UWP pour la Xbox One et vous devez donc cocher l'option Activer XBox Live dans les options de jeu UWP.


Syntaxe:

xboxlive_matchmaking_join_session(session_id);

Argument La description
session_id Le pointeur d'identification de session de la session à rejoindre.


Résultats:

DS Map


Exemple:

if !xboxlive_user_is_signed_in()
    {
    if !xboxlive_user_is_signing_in()
        {
        xboxlive_show_account_picker();
        }
    }
else
    {
    global.GamerTag = xboxlive_gamertag_for_user();
    }

Le code ci-dessus vérifie si un utilisateur est connecté à XBox Live et s’ils le sont, le code obtiendra le gamertag de l’utilisateur et sera stocké dans une variable globale pour une utilisation ultérieure. S'ils ne sont pas connectés, le code vérifie s'ils sont en train de se connecter. Dans ce cas, il ne se passera plus rien. Si ce n'est pas le cas, il ouvrira le sélecteur de compte et les invitera à signer. -dans.
