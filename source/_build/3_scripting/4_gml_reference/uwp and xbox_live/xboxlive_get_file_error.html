xboxlive_get_file_error


La description

Cette fonction renverra une constante (illustrée ci-dessous) pour aider à comprendre pourquoi une fonction de fichier a échoué afin que vous puissiez prendre les mesures appropriées dans votre code de jeu. Lorsque l'une des fonctions de fichier échoue, vous pouvez appeler la fonction pour savoir exactement quelle est l'erreur.

REMARQUE: lors de l’utilisation des fonctions de sauvegarde / chargement asynchrones, le async_load map aura une clé supplémentaire, " error " qui contiendra également l'une des constantes (voir ci-dessous).
IMPORTANT! Cette fonction est uniquement valide lors de l'exportation à l'aide de la cible UWP pour la Xbox One et vous devez donc cocher l'option Activer XBox Live dans les options de jeu UWP.


Syntaxe:

xboxlive_get_file_error();


Résultats:

Constant

Constant La description
xboxlive_fileerror_outoflocalstorage Si cela se produit lors de la sauvegarde, cela signifie que tout l'espace de sauvegarde disponible a été utilisé et que l'utilisateur a décliné l'offre (présentée par le système d'exploitation) de libérer de l'espace. Le jeu doit confirmer à l'utilisateur qu'il a l'intention de continuer à jouer sans sauvegarder. S'ils veulent toujours pouvoir sauvegarder, le jeu doit recommencer l'opération de sauvegarde.
xboxlive_fileerror_quotaexceeded Si cela se produit, le jeu doit informer l'utilisateur qu'il ne peut plus sauvegarder de données avant d'avoir libéré de l'espace, puis leur présente un moyen de le faire.
xboxlive_fileerror_noerror La fonction de sauvegarde ou de chargement de fichier a été réalisée sans problème.


Exemple:

buffer_save(buff, "SaveGame.bin"); if xboxlive_get_file_error() != xboxlive_fileerror_noerror
    {
    show_debug_message("Save failed. Error code = " + string(xboxlive_get_file_error()));
    }

Le code ci-dessus vérifie s'il y a eu une erreur d'enregistrement et si le code d'erreur est imprimé sur la console.
