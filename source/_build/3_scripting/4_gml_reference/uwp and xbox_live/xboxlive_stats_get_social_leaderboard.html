xboxlive_stats_get_social_leaderboard


La description

Cette fonction peut être utilisée pour extraire un classement social de rangs pour une statistique donnée. Vous fournissez l'ID utilisateur (tel que renvoyé, par exemple, par la fonction xboxlive_get_user ), la chaîne de statistiques (telle que définie lors de la création à l'aide des fonctions xboxlive_stats_set_stat_ *), puis vous spécifiez un certain nombre de détails sur les informations de classement que vous souhaitez récupérer. Notez que vous ne pouvez récupérer un classement social que pour des statistiques réelles ou int, mais pas pour des statistiques de chaîne. Si vous indiquez que l'argument "favourites_only" est défini sur true, les résultats ne contiendront que des données pour les amis marqués par l'utilisateur. comme "favoris".

IMPORTANT! Les statistiques utilisées dans les classements sociaux n'ont pas besoin d'être enregistrées comme "Statistiques en vedette" dans le Centre de développement XDP / Windows.


La fonction générera un rappel qui déclenchera un événement système asynchrone. Cet événement aura la carte spéciale DS async_load qui devrait ensuite être analysé pour les clés suivantes:

  • " id " - tiendra la constante achievement_stat_event

  • " eventname " - Contiendra la chaîne " GetLeaderboardComplete "

  • " userid " - L'ID utilisateur associé à la requête

  • " error " - 0 en cas de succès, une autre valeur en cas d'erreur. L'erreur la plus commune renvoyée est la suivante:

    • -2145844848: Requête incorrecte (la statistique n'est pas un type de classement valide, c'est-à-dire une chaîne)

  • " errormessage " - Chaîne avec un message d'erreur, le cas échéant

  • " display_name " - ID unique pour le classement tel qu'il est défini dans le tableau de bord du fournisseur.

  • " numentries " - Le nombre d'entrées que vous avez reçues dans le classement.

Le reste de la carte DS contiendra également les données du classement au format suivant (où "N" est la position dans les données du classement, de 0 à "numéros"):

  • " Joueur N " - Le nom du joueur, où "N" est une valeur entière correspondant à sa position dans la liste des entrées au classement.

  • " Playerid N " - L'identifiant unique du joueur, "N".

  • " Rang N " - Le rang du joueur "N" dans le classement.

  • " Score N " - Le score du joueur "N".

IMPORTANT! Cette fonction est uniquement valide lors de l'exportation à l'aide de la cible UWP pour la Xbox One et vous devez donc cocher l'option Activer XBox Live dans les options de jeu UWP.


Syntaxe:

xboxlive_stats_get_social_leaderboard(user_id, stat, num_entries, start_rank, start_at_user, favourites_only);

Argument La description
user_id L'ID utilisateur de l'utilisateur pour lequel obtenir le classement
stat La stat (sous forme de chaîne) pour créer le classement mondial à partir de
num_entries Le nombre d'entrées du classement mondial à récupérer
start_rank Le classement dans le classement de départ (utilisez 0 si l'argument "start_at_user" est défini sur true )
start_at_user Mis à true pour commencer au rang d'ID utilisateur, false sinon (réglé sur false si l'argument "start_rank" est différent de 0)
favourites_only Mis à true pour afficher uniquement les amis marqués comme "favoris" ou false autrement


Résultats:

N/A


Exemple étendu:

Voici un exemple étendu d'utilisation de cette fonction. Pour commencer, appelez-le dans un événement comme le démarrage ou la création d'une pièce:

xboxlive_stats_get_social_leaderboard(user_id, "GlobalTime", 20, 1, false, false);

Le code ci-dessus serait appelé pour obtenir une liste de toutes les positions mondiales du classement pour le jeu et générera un rappel d'événement social asynchrone que nous traiterions de la manière suivante:

if async_load[? "id"] == achievement_stat_event
    {
    if async_load[? "eventname"] == "GetLeaderboardComplete"
        {
        global.numentries = async_load[? "numentries"];
        for(var i = 0; i < numentries; i++;)
            {
            global.playername[i] = async_load[? "Player" + string(i)];
            global.playerid[i] = async_load[? "Playerid" + string(i)];
            global.playerrank[i] = async_load[? "Rank" + string(i)];
            global.playerscore[i] = async_load[? "Score" + string(i]);
            }
        }
    }

Le code ci-dessus vérifie le ds_map renvoyé dans l'événement social asynchrone et, si son "id" correspond à la constante en cours de vérification, il vérifie ensuite si l'événement a été déclenché par les données de classement renvoyées avant de parcourir en boucle la carte et de stocker toutes les valeurs différentes. dans un certain nombre de tableaux globaux.
