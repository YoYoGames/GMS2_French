uwp_check_privilege


La description

Avec cette fonction, vous pouvez vérifier qu'un utilisateur donné a un privilège choisi. La fonction renvoie true ou false dans l'événement asynchrone système, que le privilège soit activé ou non. Si vous définissez l'argument tentative_resolution sur true et que le privilège n'est pas activé, un dialogue système s'ouvrira (suspension du jeu). ) pour inviter l'utilisateur à mettre à niveau le compte ou tout autre moyen nécessaire pour obtenir le privilège. Si l'utilisateur active ensuite l'option requise, la fonction renvoie true.

La fonction nécessite que vous utilisiez l'une des constantes suivantes pour la demande de privilège:

Constant La description
uwp_privilege_internet_browsing Vérifiez si la navigation sur Internet est autorisée
uwp_privilege_social_network_sharing Vérifiez si le partage sur les réseaux sociaux est autorisé
uwp_privilege_share_kinect_content Vérifier si le partage du contrôleur Kinect est autorisé
uwp_privilege_video_communications Vérifiez si la communication vidéo est autorisée
uwp_privilege_communications Vérifiez si la communication Internet est autorisée
uwp_privilege_user_created_content Vérifier si l'accès au contenu créé par l'utilisateur est autorisé
uwp_privilege_multiplayer_sessions Vérifiez si les sessions multijoueurs en ligne sont autorisées
uwp_privilege_sessions Vérifiez si les sessions en ligne sont autorisées
uwp_privilege_fitness_upload Vérifiez si le téléchargement des données de condition physique est autorisé


Une fois la fonction appelée, elle déclenchera un événement système asynchrone à la réception du résultat du rappel, qui comportera la mappe DS spéciale. async_load. Cette carte doit ensuite être analysée pour la clé suivante:

  • " event_type " - doit contenir la valeur "check_privilege_result" si l'événement a été déclenché par cette fonction

Si le type d'événement concerne cette fonction, il y aura également les clés supplémentaires suivantes:

  • " result " - sera une ou plusieurs des constantes suivantes (valeurs combinées):
    Constant Valeur La description
    uwp_privilege_result_no_issue 0 There are no privilege issues with the user
    uwp_privilege_result_purchase_required 1 The user must purchase something additional, usually a subscription, for access
    uwp_privilege_result_aborted 2 The check was aborted
    uwp_privilege_result_banned 4 The user is banned
    uwp_privilege_result_restricted 8 The user is restricted from access, usually through parental controls

  • " privilège " - sera l'une des constantes répertoriées ci-dessus à utiliser dans la fonction. Vous pourrez ainsi déterminer le privilège que vous avez demandé en cas de vérifications multiples.


Syntaxe:

uwp_check_privilege(user_id, privilege, attempt_resolution);

Argument La description
user_id Le pointeur d'identification d'utilisateur à vérifier.
privilege La constante de privilège à vérifier (voir la description ci-dessus).
attempt_resolution Activer la tentative de résolution si défini sur true, ou ignorer si défini sur false.


Résultats:

Real


Exemple:

var user_one = xboxlive_get_user(0);
if (xboxlive_user_is_signed_in(user_one))
    {
    uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true);
    }

Ce qui précède vérifie si l’utilisateur est connecté et, s’ils le sont, vérifie l’autorisation des sessions multijoueurs.
