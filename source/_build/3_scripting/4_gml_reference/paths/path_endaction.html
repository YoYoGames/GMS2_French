path_endaction

Définit ce qu'il faut faire lorsque l'instance atteint la fin du chemin.

Syntaxe:

path_endaction;


Retourne: Constant.


La description

Cette variable peut être utilisée pour obtenir ou modifier la réaction d'une instance lorsqu'elle atteint la fin du chemin courant. Normalement, vous définissez ceci lorsque vous démarrez le chemin en utilisant path_start mais vous pouvez changer ce comportement en fonction d'un certain nombre d'événements dans votre jeu. Les valeurs disponibles sont exprimées en utilisant les constantes suivantes:

  • path_action_stop = arrêter le chemin
  • path_action_restart = continuer depuis la position de départ, sauter au début si le chemin n'est pas fermé
  • path_action_continue = recommencez le chemin à partir de la position actuelle
  • path_action_reverse = inverser la vitesse du chemin (exécuter le chemin à l'envers)


Exemple:

if path_endaction == path_action_stop
   {
   path_endaction = choose(path_action_restart, path_action_continue, path_action_reverse);
   }

Le code ci-dessus vérifiera l'action de fin de chemin et si elle est définie sur stop, une nouvelle action de fin de chemin sera sélectionnée de manière aléatoire.
