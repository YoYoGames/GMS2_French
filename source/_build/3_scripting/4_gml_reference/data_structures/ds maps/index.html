DS Maps

Une ds_map est une structure de données qui stocke les données dans des paires liées composées d'une clé et d'une valeur correspondante.

La structure de données ds_map est particulièrement utile car elle vous permet de stocker des paires clé / valeur. Par exemple, un personnage de votre jeu peut avoir plusieurs objets différents ( clés ). Pour chaque objet individuel, il peut en avoir un certain nombre ( valeurs ), comme dans un RPG où vous pouvez avoir 10 potions de santé, 5 potions de mana et 100 or. Les cartes conservent toutes ces paires au même endroit et vous pouvez les ajouter à la carte et rechercher la valeur correspondant à certaines clés à l'aide de fonctions simples. Cependant, vous devez savoir certaines choses sur les cartes avant de les utiliser! Les cartes ne sont pas triées de manière (reconnaissable), ce qui signifie que pour trouver une certaine clé, vous devrez peut-être parcourir le tout (ce qui est très lent). Il n’existe pas non plus de possibilité de conserver deux clés identiques, ni d’affecter deux valeurs à une clé.

REMARQUE: Les fonctions de ds_map ont changé depuis les versions précédentes de GameMaker. GameMaker conséquent, les jeux importés risquent de ne pas fonctionner ou de ne pas se comporter comme prévu.

REMARQUE: Comme avec toutes les ressources dynamiques, les structures de données utilisent beaucoup de mémoire. Par conséquent, elles doivent toujours être détruites lorsqu'elles ne sont plus nécessaires pour éviter les fuites de mémoire qui ralentiraient et, éventuellement, bloqueraient votre jeu.

Les fonctions suivantes existent qui traitent des cartes DS.

  1. ds_map_exists
  2. ds_map_create
  3. ds_map_add
  4. ds_map_clear
  5. ds_map_copy
  6. ds_map_replace
  7. ds_map_delete
  8. ds_map_empty
  9. ds_map_size
  10. ds_map_find_first
  11. ds_map_find_last
  12. ds_map_find_next
  13. ds_map_find_previous
  14. ds_map_find_value
  15. ds_map_set
  16. ds_map_read
  17. ds_map_write
  18. ds_map_destroy

Quatre autres fonctions sont disponibles pour enregistrer et charger un fichier ds_map. Ces fonctions vont chiffrer la carte et la stocker dans un emplacement sécurisé sur la plate-forme cible. Vous pourrez ainsi l'utiliser pour enregistrer les détails de l'achat intégré, les mots de passe, les informations de jeu sensibles, etc.

  1. ds_map_secure_save
  2. ds_map_secure_save_buffer
  3. ds_map_secure_load
  4. ds_map_secure_load_buffer

Outre ces fonctions spécifiques, vous pouvez tous utiliser une expression (appelée accesseur ) pour ajouter ou modifier le contenu de votre ds_map. Cet accesseur ressemble à un tableau 1D avec la syntaxe suivante:

map_index[? key]

Vous pouvez trouver plus d'informations, ainsi que des exemples, sur la page Vue d'ensemble de GML sur Accessors.

JSON et Ds_maps

L'une des utilisations de ds_maps est l'utilisation de JSON. Il existe donc quelques fonctions spéciales qui lui sont propres:

  1. ds_map_add_list
  2. ds_map_add_map
  3. ds_map_replace_list
  4. ds_map_replace_map

Il existe également quelques fonctions complémentaires pour ds_lists:

  1. ds_list_mark_as_list
  2. ds_list_mark_as_map

REMARQUE: Bien que ces fonctions vous permettent d'ajouter des listes et des cartes dans une carte, elles sont inutiles pour toute autre chose que JSON. Les cartes et les listes imbriquées ne seront pas lues correctement si elles sont écrites sur le disque ou utilisées d'une autre manière.

REMARQUE: si vous devez vérifier si une structure de données existe, vous pouvez utiliser le ds_exists() une fonction.
