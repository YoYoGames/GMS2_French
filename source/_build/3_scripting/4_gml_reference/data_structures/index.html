Structures de données


Dans les jeux, vous devez souvent stocker les informations de manière précise et ordonnée. Par exemple, vous pouvez avoir besoin de stocker des listes d’articles transportés par une personne ou de stocker une grille d’endroits qui doivent encore être visités. Vous pouvez maintenant utiliser les tableaux pour cela, mais si vous souhaitez effectuer des opérations plus complexes, telles que le tri des données ou la recherche d'un élément particulier, vous devez écrire de gros morceaux de code GML qui peuvent être lents à s'exécuter, difficiles à déboguer. et une vraie douleur à écrire!

Pour remédier à cela, GameMaker Studio 2 possède un certain nombre de structures de données intégrées accessibles via des fonctions spécialisées. Il existe six types de structure de données disponibles, chacun ayant ses propres avantages en fonction du type d'informations que vous souhaitez stocker et de la manière dont vous souhaitez les manipuler ultérieurement: piles, files d'attente, listes, cartes, priorités et grilles..

Toutes les structures de données fonctionnent de la même manière: vous créez une structure de données et stockez sa valeur d'index dans une variable. Vous utilisez ensuite cet index pour référencer la structure de données dans tous les autres appels de fonction pouvant effectuer des opérations dessus. Enfin, une fois que vous avez terminé, vous détruisez à nouveau la structure de données pour la supprimer de la mémoire. Vous pouvez utiliser autant de structures en même temps que vous le souhaitez et toutes les structures peuvent stocker des chaînes et des valeurs réelles.

REMARQUE: Comme avec toutes les ressources dynamiques, les structures de données utilisent beaucoup de mémoire. Par conséquent, elles doivent toujours être détruites lorsqu'elles ne sont plus nécessaires pour éviter les fuites de mémoire qui ralentiraient et, éventuellement, bloqueraient votre jeu.
IMPORTANT! Lorsque vous créez une structure de données, la valeur d'index identifiant celle-ci est un entier commençant à 0. Cela signifie que différentes structures de données peuvent avoir la même valeur d'index. Par conséquent, en cas de doute, utilisez la ds_exists fonctionner avant d'y accéder. Notez également que les index sont réutilisés. Ainsi, une valeur d'index de structure de données détruite peut être utilisée ultérieurement par un indicateur nouvellement créé. Nous vous recommandons donc de définir toute variable contenant un index DS à -1 après avoir détruit la structure de données.

Avant d'utiliser des structures de données dans votre jeu, vous devez également savoir qu'il existe certains moments où, en raison d'erreurs d'arrondi, vous pouvez obtenir un résultat qui ne correspond pas à vos attentes. Cela peut être résolu en modifiant la précision DS à l'aide de la fonction suivante:

  • ds_set_precision

Il existe également une fonction spéciale permettant de vérifier si une structure de données d'un type donné existe:

  • ds_exists
