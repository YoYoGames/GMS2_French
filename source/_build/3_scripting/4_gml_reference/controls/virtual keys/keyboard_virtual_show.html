keyboard_virtual_show


La description

Cette fonction peut être utilisée pour afficher le clavier virtuel sur l'appareil exécutant le jeu. Lorsque vous appelez cette fonction, vous devez fournir l'une des constantes suivantes pour chacun des trois premiers arguments:

  • Type de clavier: Cette constante permet de définir quel clavier sera disponible sur le clavier virtuel. Les types de clavier disponibles sont:
    Type de clavier La description
    kbv_type_default Le type de clavier par défaut pour le système actuel.
    kbv_type_ascii Un clavier uniquement ASCII.
    kbv_type_url Un clavier normal optimisé pour la saisie d'URL. Comporte généralement un ".com" ou d'autres clés de domaine, ainsi que "/" et "." clés.
    kbv_type_email Un clavier normal optimisé pour la saisie de courrier électronique. Caractéristiques généralement "@" et "." personnages.
    kbv_type_numbers Un clavier composé uniquement de chiffres, généralement affiché sous forme de pavé numérique.
    kbv_type_phone Un clavier de téléphone. Généralement, chiffres uniquement avec les touches "*" et "#".
    kbv_type_phone_name Un clavier optimisé pour la saisie d’un numéro de téléphone et d’un nom. Généralement similaire à un clavier ASCII, mais avec une sélection limitée de caractères spéciaux.

  • Type de retour: Cette constante permet de définir ce qui est affiché sur la touche retour / action du clavier virtuel. Les types de retour disponibles sont:
    Type de retour La description
    kbv_returnkey_default Le titre de clé de retour par défaut pour le système actuel.
    kbv_returnkey_go Définit le titre de la clé de retour sur "Go".
    kbv_returnkey_google Définit le titre de la clé de retour sur "Google" ou sur une icône de recherche générique dans certains cas.
    kbv_returnkey_join Définit le titre de la clé de retour sur "Go".
    kbv_returnkey_next Définit le titre de la clé de retour sur "Suivant".
    kbv_returnkey_route Définit le titre de la clé de retour sur "Route".
    kbv_returnkey_search Définit le titre de la clé de retour sur "Rechercher" ou sur une icône de recherche générique dans certains cas.
    kbv_returnkey_send Définit le titre de la clé de retour sur "Envoyer".
    kbv_returnkey_yahoo Définit le titre de la clé de retour sur "Yahoo" ou sur une icône de recherche générique dans certains cas.
    kbv_returnkey_done Définit le titre de la clé de retour sur "Terminé".
    kbv_returnkey_continue Définit le titre de la clé de retour sur "Continuer".
    kbv_returnkey_emergency Définit le titre de la touche de retour sur "Appel d'urgence".

  • Type de capitalisation automatique: Cette constante permet de déterminer comment les mots saisis via le clavier virtuel doivent être autocapitalisés. Les types d'autocapitalisation disponibles sont:
    Type d'autocapitalisation La description
    kbv_autocapitalize_none L'autocapitalisation est désactivée.
    kbv_autocapitalize_words Les mots seront auto-capitalisés.
    kbv_autocapitalize_sentences Les phrases seront auto-capitalisées.
    kbv_autocapitalize_characters Tous les personnages seront en majuscule.

Le dernier argument consiste à activer / désactiver le texte prédictif, ce qui serait défini sur true pour le permettre, et false sinon, notez que le fait que ce soit autorisé ne signifie pas qu'il sera utilisé, car cela dépendra des préférences de l'utilisateur sur l'appareil. En mode de texte prédictif, le clavier virtuel ne génère pas d'événements de pression de touche GameMaker Studio 2 normaux. Au lieu de cela, il ne mettra à jour que le dernier caractère saisi et les variables de chaîne du clavier. Cela est dû à l'impossibilité de détecter si une modification du champ de texte interne utilisé pour détecter les appuis sur les touches provient d'une touche réelle du clavier virtuel ou d'une suggestion de texte. Dans ces cas, vous voudriez lire le keyboard_string entrée par opposition à la lecture de tout type d’entrée de clé brute.

Il est également important de noter que l'utilisateur obtiendra différents claviers avec des fonctionnalités différentes selon le système d'exploitation de la plate-forme, avec les réserves suivantes à utiliser par cible:

  • UWP: l'implémentation UWP ne prend pas en charge return_key_type, autocapitalization_type ou predictive_text_enabled. Transmettre ces paramètres dans une fonction ne causera aucune erreur, ils n'auront aucun effet sur le clavier.

  • Android / Amazon Fire: Sous Android, le type de clavier kbv_type_phone_name n'est pas pris en charge et affiche un clavier ASCII à la place, et le type de clé de retour ne peut être que kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next, kbv_returnkey_send ou kbv_returnkey_done. Si d'autres types de clé de retour sont utilisés sur cette plate-forme, le système utilisera par défaut les clés de retour de remplacement répertoriées ci-dessous:
    Clé de retour indisponible Clé de remplacement
    kbv_returnkey_google
    kbv_returnkey_yahoo     		kbv_returnkey_search
    kbv_returnkey_join
    kbv_returnkey_route
    kbv_returnkey_emergency     		kbv_returnkey_go
    kbv_returnkey_continue kbv_returnkey_next


  • Android TV / FireTV: Les types de touche de retour personnalisés ne sont pas pris en charge sur les claviers ASCII - la touche de retour par défaut sera toujours affichée.

L’appel de cette fonction générera un événement asynchrone système, dans lequel le async_load La carte DS sera remplie avec les paires clé / valeur suivantes:

  • " event_type " - ce sera "l' état du clavier virtuel " lorsqu'il est déclenché par des fonctions de clavier virtuel.

  • " screen_height " - la hauteur du clavier virtuel (en pixels). Ce sera 0 si le clavier est invisible.

  • " keyboard_status " - l'état actuel du clavier, renvoyé sous l'une des chaînes suivantes:

    • "cache"
    • "caché"
    • "montrer"
    • "visible"


Syntaxe

keyboard_virtual_show(keyboard_type, return_key_type, autocapitalization_type, predictive_text_enabled);

Argument La description
keyboard_type Détermine le jeu de clés disponible sur le clavier virtuel.
return_key_type Détermine ce qui est affiché sur la touche retour / action du clavier virtuel.
autocapitalization_type Détermine comment / si les mots saisis via le clavier virtuel seront autocapitalisés.
predictive_text_enabled Mis à true / false activer / désactiver la saisie de texte intuitive.


Résultats:

N/A


Exemple:

if input == false
    {
    input = true;
    keyboard_virtual_show(kbv_type_numbers, kbv_returnkey_default, kbv_autocapitalize_none, false);
    }

Le code ci-dessus fera apparaître le clavier virtuel du système d'exploitation si la variable donnée n'est pas définie sur true.
