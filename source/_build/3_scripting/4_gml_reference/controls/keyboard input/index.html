La saisie au clavier


Lorsque vous GameMaker Studio 2 le clavier dans GameMaker Studio 2 vous disposez de nombreuses fonctions permettant de reconnaître différents états du clavier, tels que pressé ou relâché. Il y en a aussi qui stockent toutes les touches pressées sous forme de chaîne ou qui peuvent vous dire quelle était la dernière touche pressée, ainsi que d'autres qui vous permettent d'effacer complètement l'état du clavier.

REMARQUE: Ces fonctions sont conçues pour les plates-formes de bureau Windows / Mac / Ubuntu uniquement. Vous constaterez peut-être que certaines des variables et constantes intégrées ne sont pas valides sur d'autres plates-formes et que de nombreuses fonctions ne fonctionnent pas sur les mobiles.


Chaque caractère saisi à partir d'une clé (ou de plusieurs clés) est défini par son code UTF8, qui est une valeur numérique. Cette valeur peut être récupérée pour n’importe quel caractère en utilisant le ord() fonction, mais GameMaker Studio 2 propose également une série de constants pour le clavier le plus utilisé des touches spéciales et des fonctions spéciales. En règle générale, vous utiliseriez une combinaison de ord() avec le keyboard_check*() fonctions, quelque chose comme ceci:

if keyboard_check(ord("A"))
    {
    hspeed = -5;
    }


Donc, ce qui précède vérifiera la touche "A" et si vous appuyez dessus, la vitesse horizontale de l'objet sera réglée sur -5. Notez que l'utilisation de ord() de cette manière ne fonctionnera correctement que si la chaîne en entrée est composée d'un seul caractère et d'un nombre compris entre 0 et 9 ou d'un caractère romain majuscule de A à Z. La fonction ord() retournera une valeur UTF8 complète, mais le keyboard_check*() les fonctions ne détecteront que A - Z et 0 - 9. Mais que faire si vous voulez utiliser les touches de direction? Ou si vous souhaitez modifier une action en utilisant la touche "shift"? Eh bien, pour cela, GameMaker Studio 2 propose une série de vk_ constantes ( vk_ représente un clavier virtuel ) que vous pouvez utiliser à la place de ord():

Les constantes suivantes ne peuvent être utilisées qu'avec keyboard_check_direct():

Constant La description
vk_lshift touche Maj gauche
vk_lcontrol touche de contrôle gauche
vk_lalt touche alt gauche
vk_rshift touche Maj droite
vk_rcontrol touche de contrôle droite
vk_ralt touche alt droite
Ce qui suit est un petit exemple de la façon d’utiliser le vk_ constantes:

if keyboard_check_pressed(vk_tab)
    {
    instance_create_layer(x, y, "Controllers", obj_Menu);
    }


Le code ci-dessus détectera si la touche "Tab" est enfoncée et créera une instance d'objet "obj_Menu" si c'est le cas.

Si vous devez rechercher un caractère clé différent de 0 à 9, de A à Z ou de l'une des constantes VK, vous devez alors vérifier l'une des keyboard_* des variables, comme keyboard_lastchar par exemple:
