Fonctions numériques


Les nombres réels dans GameMaker Studio 2 sont considérés comme des nombres à virgule flottante double précision, c'est-à-dire qu'ils ont un point décimal, sans nombre fixe de chiffres, ni avant ni après le point. 2, par exemple, est un entier binaire mais 2.0 est un réel à virgule flottante. Cette distinction entre nombres entiers et flottants est très importante pour le développement multiplateforme, car la précision des calculs effectués sur un PC Windows n’est pas la même que celle de ces mêmes calculs lorsqu’il est effectué sur un appareil mobile. Cela signifie que vous devez porter une attention particulière lors de la comparaison, par exemple:

if image_index == 1 {do something...}

Cela peut sembler fonctionner, mais en raison de la façon dont fonctionne le calcul en virgule flottante, il se peut que cela signifie que image_index est égal à 1.00000000001. Dans ce cas, la comparaison ne donnera jamais true résultat et le code en cours d'exécution. Ces types d’erreurs peuvent être assez difficiles à corriger et il est donc toujours bon d’en être conscient, d’avoir recours à d’autres moyens pour vérifier les valeurs ou d’utiliser les fonctions de revêtement de sol ou d’arrondissement appropriées (énumérées ci-dessous) pour convertir le nombre en valeurs. vérifier dans un entier. Par exemple, le code ci-dessus pourrait être écrit comme suit:

if floor(image_index) == 1 {do something...}

Il est également intéressant de noter que lors de l'utilisation des cibles YoYo Compiler, toutes les expressions et fonctions sont évaluées de gauche à droite, tandis que sur toutes les autres plates-formes cibles, elles sont évaluées de droite à gauche, ce qui signifie que cela donnera par exemple des résultats différents Sur la plateforme:

val = max(num, ++num, num++);

Vous pouvez également utiliser une fonction spéciale disponible dans GameMaker Studio 2 pour définir la valeur epsilon pour les calculs en virgule flottante. Lorsqu'un nombre réel est arrondi au nombre en virgule flottante le plus proche, epsilon (également appelé "machine epsilon") forme une limite supérieure de l'erreur relative. Vous pouvez obtenir et définir la valeur epsilon à l'aide des fonctions suivantes:

  • math_set_epsilon
  • math_get_epsilon

REMARQUE: lors de l'utilisation des fonctions aléatoires, GameMaker Studio 2 conserve la même graine aléatoire à chaque démarrage du jeu. Cela facilite beaucoup le débogage, car vous êtes assuré que les fonctions aléatoires renverront toujours la même valeur. Toutefois, si vous ne le souhaitez pas, vous devez d'abord définir une nouvelle graine aléatoire au tout début du jeu, soit en utilisant randomise ou random_set_seed.
