Débogage


Lorsque vous créez un jeu, il est souvent très important que vous connaissiez certains détails de la mécanique du jeu à des moments précis dans le temps ou que vous puissiez saisir certaines valeurs clés pour certaines fonctions afin de tester différentes méthodes et idées pour votre jeu. À cette fin, GameMaker Studio 2 fournit un certain nombre de fonctions de débogage qui affichent des messages et demandent des entrées en mode débogage (ou sur une cible Windows), mais qui seront ignorées lors de la compilation pour d’autres modules cible, car les fonctions mettent le jeu en boucle fermée jusqu'à la résolution de l'entrée.

REMARQUE: bien qu'ils puissent fonctionner sur certaines plates-formes, ils ne doivent être utilisés que pour le débogage.
  • Mode débogage
  • debug_event
  • debug_get_callstack
  • get_integer
  • get_string
  • show_error
  • voir le message
  • show_question
  • show_debug_message
  • show_debug_overlay
  • code_is_compilé
  • fps
  • fps_real

Outre ces fonctions spécialisées, GameMaker Studio 2 contient également une longue liste de messages d'erreur pouvant être affichés lors de la compilation de votre jeu ou lors de son exécution. Ces messages indiquent l'erreur spécifique en question et peuvent être utilisés pour identifier un problème dans un objet ou un script spécifique, ce qui facilite grandement le processus de débogage de votre jeu. Toutefois, la signification exacte du message d'erreur n'est pas toujours évidente.
