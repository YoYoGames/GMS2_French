mp_grid_add_instances

Syntaxe:

mp_grid_add_instances(id, obj, prec);


Argument La description
id Index du mp_grid à utiliser
obj Index d'objet, ou id d'instance, des instances à ajouter dans mp_grid
prec Indique si la vérification est basée sur des collisions au pixel près (vrai = lent) ou sur son cadre de sélection en général (faux = rapide).


Retours: N / A


La description

Cette fonction utilise le masque de collision d'instance (déterminé par les propriétés de l'image-objet ou l'index de masque de l'instance appelante) pour marquer les cellules comme interdites ou non, les cellules interdites ne pouvant être croisées par aucune des fonctions de recherche de chemin. Vous pouvez spécifier dans la fonction si vous souhaitez ou non prendre en compte des collisions précises. Les cellules interdites marquées changeront en fonction de ce paramètre. Cette image illustre ce comportement:

Les deux instances ci-dessus ont été ajoutées dans mp_grid à l'aide de la fonction mp_grid_add_instances avec la valeur précise définie comme true. Comme vous pouvez le voir, l'instance verte a uniquement marqué l'interdiction des carrés de la grille qu'elle "touche", car son masque d'image-objet est également défini avec précision. Cependant, la deuxième instance a également marqué d'autres cellules qui ne semblent pas se toucher. En effet, le masque de l'image-objet n'est pas défini avec précision, ce qui signifie que même si vous utilisez le paramètre précis de la fonction, seul le cadre de sélection de l'image-objet sera pris en compte. C’est également ce qui se produirait si vous définissez l’argument précis de la fonction sur false. Même les occurrences avec un masque de sprite précis seront ajoutées à la grille en fonction de leurs cadres de sélection.


Exemple:

mp_grid_add_instances(grid, obj_Wall, true);

Le code ci-dessus ajoutera toutes les occurrences de "obj_Wall" dans la mp_grid indexée dans la variable "grid" en utilisant le masque de collision précis plutôt que le cadre de sélection.
