get_integer_async


La description

Cette fonction ouvre une fenêtre et affiche un message ainsi qu'un espace pour que l'utilisateur entre une valeur (qui contiendra la valeur par défaut fournie pour commencer). Il s’agit d’une fonction asynchrone et, en tant que telle, GameMaker Studio 2 ne bloque pas le périphérique sur GameMaker Studio 2 elle est en cours d’exécution en attendant la réponse, mais continue à exécuter les événements normalement. Une fois que l'utilisateur a tapé sa chaîne et appuyé sur le bouton "OK", un événement de dialogue asynchrone est déclenché pour lequel, pour la durée de cet événement uniquement, un ds_map est stocké dans la variable. async_load.

Cette carte contiendra les trois clés, " id "," status ", et " value "." id "est la valeur renvoyée par la fonction lors de l'appel, le" status "sera soit true pour appuyer sur le bouton "OK", ou false si le message a été annulé (le cas échéant, toutes les plateformes cibles n'autorisant pas l'annulation des messages). Finalement " value "retournera le nombre entier saisi par l’utilisateur sous forme de chaîne (une chaîne vide" "sera renvoyée si aucune entrée n’a été donnée).


Syntaxe:

get_integer_async(string, default);

Argument La description
String Le message à afficher dans la boîte de dialogue.
default La valeur par défaut


Résultats:

Real


Exemple étendu:

L' événement de pression de souris gauche (par exemple) de l'objet qui affiche le message aurait le code suivant:

msg = get_integer_async("How old are you?", 0);

Ce qui précède montrera un message demandant à l’utilisateur de saisir une chaîne et d’appuyer sur "OK". La fonction id est stockée dans la variable "msg" et sera utilisée dans l' événement asynchrone Dialogs comme indiqué ci-dessous:

var i_d = ds_map_find_value(async_load, "id");
if i_d == msg
   {
   if ds_map_find_value(async_load, "status")
      {
      global.Age = ds_map_find_value(async_load, "value");
      }
   }

Le code ci-dessus vérifie le " id "clé du rendu ds_map par rapport à la valeur stockée dans la variable "msg". S'ils sont identiques, il vérifie ensuite si "OK" a été activé (plutôt que la fenêtre en cours de fermeture / annulée) et si elle est renvoyée. true il définit ensuite une variable globale à partir du nombre entier renvoyé.
