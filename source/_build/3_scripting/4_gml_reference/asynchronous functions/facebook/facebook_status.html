statut Facebook


Syntaxe:

facebook_status()


Résultats:

String


La description

Avec cette fonction, vous pouvez interroger Facebook pour connaître l'état actuel de la connexion de l'utilisateur. Une des chaînes suivantes est renvoyée (notez qu'elles sont toujours renvoyées en lettres majuscules):

  • AUTHORISED: Cela signifie que l'utilisateur s'est connecté correctement, mais toutes les autorisations n'ont peut-être pas été accordées.
  • PROCESSING: Ceci est renvoyé lorsque le processus de connexion est actuellement non résolu.
  • IDLE: Ceci est renvoyé lorsque la connexion à l'API Facebook a été initialisée mais qu'aucune autre action n'a été entreprise (c'est-à-dire qu'aucun utilisateur n'est connecté).
  • FAILED: Cela signifie que l'utilisateur ne s'est pas connecté correctement, généralement à cause d'une erreur de connexion.

Notez que cette fonction ne vérifie pas en permanence l'API Facebook et ne sera mise à jour qu'une fois qu'une action Facebook a été résolue (comme se connecter, se déconnecter ou demander des autorisations).


Exemple:

switch (facebook_status())
   {
   case "AUTHORISED": global.Auth = true; instance_destroy(); break;
   case "FAILED": facebook_login(permissions); alarm[0] = 30; break;
   default: alarm[0] = 30;
   }

Le code ci-dessus serait exécuté dans une alarme et vérifie le statut de Facebook pour voir si l'utilisateur s'est connecté correctement. Si la chaîne de retour est "AUTHORISED", une variable globale est définie sur true et le jeu continue. Si la chaîne de retour est "FAILED" le jeu demande à l'utilisateur de se reconnecter puis de répéter l'alarme pour vérifier l'état de nouveau..
