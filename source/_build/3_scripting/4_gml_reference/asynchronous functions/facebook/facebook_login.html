Identifiant Facebook


Syntaxe:

facebook_login(permissions, ios_login_type)

Argument La description
permissions Les autorisations que vous souhaitez obtenir de l'utilisateur qui se connecte.
ios_login_type Le type de connexion iOS uniquement (voir la liste des constantes ci-dessous).


Résultats:

N/A


La description

Avec cette fonction, GameMaker Studio 2 se connecte à Facebook et demande à l'utilisateur de se connecter à son compte. Vous pouvez également fournir des demandes d’autorisations supplémentaires sous la forme d’une liste_ds que cette fonction convertit ensuite en un fichier. json tableau pour les communiquer à Facebook. Vous pouvez demander une grande variété d'autorisations. Elles sont documentées sur les pages de développeurs Facebook que vous trouverez ici. Si vous ne souhaitez pas demander d'autorisations spéciales ou étendues, vous devrez toujours envoyer une liste_ds, mais cette fois, elle sera vide.

Cette fonction est également considérée comme étant asynchrone, car tant que la fenêtre de connexion de Facebook est ouverte à l’écran, le jeu continue de fonctionner en arrière-plan et même lorsque l’utilisateur a terminé la procédure de connexion, aucune des autres fonctions de Facebook ne fonctionnera tant que la facebook_status() renvoie "AUTORISE". En raison de la nécessité d'une connexion Internet et des taux de transfert de données différents, cela signifie que plusieurs secondes peuvent s'écouler avant la réception de cette autorisation et que le jeu continue à fonctionner.

REMARQUE: Vous pouvez désormais utiliser uniquement les autorisations de lecture par défaut pour cette fonction. Si vous avez besoin d'autorisations d'écriture ou d'autorisations de lecture étendues, vous devez appeler facebook_request_publish_permissions() après vous être connecté. Notez également que si votre jeu requiert plus que public_profile, email et user_friends, il devra être examiné par Facebook avant de pouvoir être utilisé par des personnes autres que les développeurs du jeu.


Un paramètre supplémentaire est requis pour la connexion iOS (les autres plates-formes peuvent simplement utiliser fb_login_default pour l'argument). Cela devrait être une des constantes listées ci-dessous:

Constant La description
fb_login_default Utilisez le comportement de connexion iOS par défaut (voir ici pour plus de détails).
fb_login_fallback_to_webview Essayez de vous connecter à Facebook en demandant à l'utilisateur des informations d'identification si nécessaire.
fb_login_no_fallback_to_webview Essayez de vous connecter à Facebook, mais aucune demande directe d’identifiants ne sera faite.
fb_login_forcing_webview Essayez uniquement de vous connecter à WebView, en demandant à l'utilisateur des informations d'identification.
fb_login_use_system_account Essayez de vous connecter à Facebook en préférant le compte système et en basculant vers un commutateur d'application rapide si nécessaire.
fb_login_forcing_safari Essayez uniquement de vous connecter avec Safari.

Les constantes répertoriées ci-dessus modifient le comportement de connexion sur iOS, vous permettant de sélectionner les méthodes de secours à utiliser (le cas échéant) en cas d'échec de la connexion via le processus normal. Pour plus d'informations, veuillez consulter cette page de développeur Facebook.


Exemple:

facebook_init();
var permissions = ds_list_create();
ds_list_add(permissions,"public_profile", "user_friends");
facebook_login(permissions, fb_login_default);
ds_list_destroy(permissions);

Le code ci-dessus initialisera Facebook, puis connectera l'utilisateur actuel, en demandant les autorisations de lecture données.
