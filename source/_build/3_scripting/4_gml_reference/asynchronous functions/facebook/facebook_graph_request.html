facebook_graph_request


Syntaxe:

facebook_graph_request(graph_path, httpMethod, ds_map_parameters, ds_map_return)

Argument La description
graph_path La partie du graphe social avec laquelle interagir
httpMethod La méthode http à utiliser ( POST, GET, DELETE ).
ds_map_parameters Le ds_map avec les informations à envoyer.
ds_map_return Le ds_map à recevoir (-1 pour aucun).


Résultats:

N/A


La description

Avec cette fonction, l'utilisateur peut interagir avec le graphique social de Facebook. L'argument "graph_path" est l'endroit où vous définissez la partie du graphique à laquelle vous souhaitez accéder, telle que la liste d'amis de l'utilisateur actuel, ou les commentaires d'une autre application ou même un événement. Le chemin exact peut être défini à l'aide des termes décrits dans la section Publication des pages de l' App Graph Graph de Facebook. Vous définissez ensuite la méthode http à utiliser, généralement POST ou GET mais Facebook accepte également la méthode DELETE.

L’argument suivant est légèrement plus complexe car il nécessite que vous ayez créé et rempli une ds_map avec les informations correctes que GameMaker Studio 2 convertira ensuite en json automatiquement lors de l'envoi à l'API Facebook. Les informations que vous mettez dans cette carte dépendront beaucoup du chemin que vous choisissez d'utiliser et une liste complète de toutes les valeurs possibles peut être trouvée ici. Vous trouverez un exemple de la structure de cette carte dans l'exemple de code fourni ci-dessous.

Enfin, nous avons un argument pour stocker toutes les informations que Facebook a renvoyées à partir de la demande de graphique. Cette information se présente sous la forme de json lequel GameMaker Studio 2 transforme en ds_map. Pour que cela fonctionne correctement, vous devez avoir préalablement créé un fichier ds_map. Si cette carte a déjà été utilisée ailleurs et qu'elle contient déjà des paires clé / valeur, elle sera effacée par cette fonction avant l'ajout des données Facebook. Il est très important que vous lisiez la documentation de Facebook sur les valeurs de retour possibles afin de savoir exactement à quoi vous attendre. Heureusement, Facebook a une page de test Graph Api où vous pouvez vérifier différents chemins de graphes et voir exactement quelles informations ils vont renvoyer.

REMARQUE: L'utilisateur doit être connecté pour que la demande de graphique puisse fonctionner.


Exemple:

var wallPostMap;
wallPostMap = ds_map_create();
ds_map_add(wallPostMap, "message", "I just got a hi-score playing Catch The Clown!");
ds_map_add(wallPostMap, "picture", "http://MacSweeneygames.com/Clown.jpg");
ds_map_add(wallPostMap, "link", "http://MacSweeneygames.com/");
ds_map_add(wallPostMap, "name", "Catch The Clown");
ds_map_add(wallPostMap, "caption", "MacSweeneygames.com");
ds_map_add(wallPostMap, "description", "Play Catch the clown now on MacSweeney Games!");
facebook_graph_request("me/feed", "POST", wallPostMap, -1);
ds_map_destroy(wallPostMap);

Le code ci-dessus va créer un fichier ds_map et stocker son index dans la variable locale "wallPostMap". Il remplit ensuite ds_map avec les paires clé / valeur appropriées nécessaires pour publier une image et du texte sur le mur des utilisateurs actuellement connectés. Enfin, il envoie la demande de graphique et supprime le fichier ds_map de la mémoire.
