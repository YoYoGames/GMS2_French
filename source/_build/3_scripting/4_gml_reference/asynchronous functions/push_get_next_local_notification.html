push_get_next_local_notification


La description

Cette fonction remplira un pré-fait ds_map avec une série de paires clé / valeur pour la prochaine notification push locale dans la file d'attente à afficher. Il renvoie -1 s'il n'y a pas d'autres notifications en file d'attente ou une valeur réelle représentant autrement l'ID de la notification. Cet identifiant peut ensuite être utilisé pour annuler la notification à l'aide de la fonction push_cancel_local_notification(). Vous devriez appeler la fonction push_get_first_local_notification pour obtenir la première notification dans la file d'attente, puis utilisez cette fonction pour continuer.

Ds_map contiendra les paires clé / valeur suivantes:

  • " title " - Le titre de la notification (disponible uniquement sur la Android forme Android, car iOS ne stocke pas les informations relatives au titre).
  • " message " - Le texte du corps du message
  • " data " - La chaîne du paquet de données
REMARQUE: cette fonction est limitée aux modules cibles iOS et Android.


Syntaxe:

push_get_next_local_notification(map);

Argument La description
map le ds_map ID à utiliser.


Résultats:

Real


Exemple:

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
   {
   var data = ds_map_find_value(map, "data");
   if data == "Daily_Reward"
      {
      push_cancel_local_notification(ntf);
      }
   ntf = push_get_next_local_notification(map);
   }
ds_map_destroy(map);

Le code ci-dessus crée un ds_map puis demande des données pour la première notification disponible dans la file d'attente. Si une notification existe, la carte est remplie et le code vérifie ensuite la clé " data " pour une chaîne spécifique. Si la chaîne est trouvée, la notification est annulée et passe à la prochaine notification jusqu'à ce que toutes les personnes de la file d'attente aient été vérifiées.
