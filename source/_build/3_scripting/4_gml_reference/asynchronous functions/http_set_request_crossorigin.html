http_set_request_crossorigin


La description

Avec cette fonction, vous pouvez définir le type d’origine croisée à utiliser lors du chargement d’images à partir d’un fichier. Cette fonction est exclusivement destinée à la plate-forme HTML5 et n’est utile que lors du chargement des sprites à partir d’un fichier à l’aide de la commande. sprite_add une fonction. Notez que lorsqu'il est défini sur "use-credentials", vous n'avez plus besoin de donner un chemin absolu à l'image-objet en cours de chargement, mais plutôt un chemin relatif (comme illustré dans l'exemple ci-dessous). Par défaut, le type d'origine croisée est défini sur "anonyme".


Syntaxe:

http_set_request_crossorigin(origin_type);

Argument La description
origin_type Le type d'origine croisée à utiliser (une chaîne)


Résultats:

N/A


Exemple:

if string_lower(http_get_request_crossorigin()) != "use-credentials"
    {
    http_set_request_crossorigin("use-credentials");
    }
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

Le code ci-dessus vérifie d'abord le type d'origine croisée actuellement défini et s'il ne s'agit pas de "use-credentials", il est défini sur "use-credentials", puis un sprite est ajouté à partir d'un fichier.
