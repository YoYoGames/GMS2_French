sprite_add


Syntaxe:

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

Argument La description
fname Le nom (un chemin de fichier de chaîne) du fichier à ajouter.
imgnum Utilisez pour indiquer le nombre de sous-images (1 pour une seule image ou pour un *.gif ).
removeback Indique si tous les pixels avec la couleur d'arrière-plan (pixel en bas à gauche) doivent être transparents.
smooth Indique si les bords doivent être lissés s'ils sont transparents.
xorig Indique la position x de l'origine dans le sprite.
yorig Indique la position y de l'origine dans le sprite.


Résultats:

Real (sprite ID value)


La description

Avec cette fonction, vous pouvez ajouter une image en tant qu’image-objet, en la chargeant depuis une source externe où le fichier image à charger doit toujours se trouver dans l’un ou l’autre *.png, *.gif ou *.jpg/jpeg format. La fonction renvoie le nouvel index de l'image-objet, qui doit ensuite être utilisé dans tous les codes ultérieurs associés à l'image-objet. Si vous utilisez cette fonction avec HTML5 ou si vous obtenez une image à partir d'une URL, cette fonction générera également un événement asynchrone Image Loaded. Voir la page Evénement asynchrone pour plus d'informations.

Toutes les images à transformer en images-objets animées, à l’exception de *.gif (voir ci-dessous) - devrait avoir un format "bande" (voir l'image ci-dessous) et sera divisé en le nombre de sous-images spécifiées suivant la règle largeur de l'image-objet = largeur de la bande / sous-images.

Comme vous pouvez le voir dans l'image ci-dessus, l'image-objet a été placée sur un fond violet foncé. Cette opération peut être supprimée en définissant l'argument "removeback" sur true. Cela fonctionne en vérifiant la couleur à cet endroit dans le pixel en bas à gauche de l’image-objet, puis l’utilise comme couleur à supprimer. Par exemple, dans l'image ci-dessus, si la couleur du pixel en bas à gauche était verte, toutes les parties vertes de l'image-objet auraient été supprimées et le reste de l'arrière-plan violet aurait été ignoré.

Si vous choisissez l'option "removeback", vous pouvez également souhaiter que GameMaker Studio 2 les bords de l'image-objet en définissant l'argument "smooth" sur true. Cela ne fait que créer une bordure semi-transparente autour des bords de l'image-objet après la suppression de son arrière-plan.

Enfin, vous pouvez également spécifier les origines x et y du sprite. Il s’agit du point où l’image-objet est "fixée" sur l’instance qui l’utilise, et est toujours calculée comme étant relative par rapport au coin supérieur gauche 0,0 de la sous-image d’une image-objet. Ainsi, par exemple, une image-objet de 32 x 32 pixels dont les valeurs sont définies sur (16,16) aura son origine au centre.

Si vous ajoutez un *.gif Pour formater l’image, il vous suffit de spécifier le nom du fichier et le numéro de l’image (qui doit être égal à 1 pour la première image) et laisser tous les autres arguments à 0, sachant que seule la première image du fichier GIF sera chargée.

Par défaut, toutes les nouvelles images-objets voient leur boîte englobante calculée automatiquement (la boîte englobante exacte dépend de la taille et de la transparence de l'image-objet). Vous pouvez toutefois personnaliser cette option, auquel cas vous devriez également utiliser la fonction. sprite_collision_mask.

REMARQUE: lorsque vous chargez une image-objet dans GameMaker Studio 2 vous devez vous rappeler de la supprimer (avec sprite_delete ) lorsqu’on n’en a plus besoin, sinon il y a un risque de fuite de mémoire qui ralentira et finira par planter votre partie.
REMARQUE: sur iOS, si vous chargez un fichier inclus à partir d'un dossier, c.-à-d.: Avec un chemin similaire à " Backgrounds/background1.png ", vous n'avez PAS besoin d'inclure le dossier dans le chemin et il vous suffit de fournir le nom du fichier. Sur toutes les autres plates-formes, vous devez fournir le (s) dossier (s) dans le chemin du fichier.
REMARQUE: En HTML5, si vous chargez une image-objet à partir d'un serveur sécurisé, vous devrez peut-être définir le type d'origine croisée et utiliser un chemin relatif au lieu d'un chemin absolu. Voir http_set_request_crossorigin pour plus de détails.
REMARQUE: Si vous chargez l'image-objet à partir d'une URL, le chargement se fera de manière asynchrone et déclenchera un événement asynchrone Image Loaded.
IMPORTANT! Cette fonction n'est pas disponible avec la licence d'évaluation du produit.


Exemple:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);

Le code ci-dessus charge un sprite dans le jeu et stocke son index dans la variable "spr".
