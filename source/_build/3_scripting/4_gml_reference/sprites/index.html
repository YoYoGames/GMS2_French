Sprites


Les sprites sont généralement les représentations visuelles des objets dans le jeu que vous créez, bien qu'ils puissent être utilisés à de nombreuses autres fins. Une image-objet est soit une image unique, soit un ensemble d'images qui, lues l'une après l'autre, ressemble à une animation en boucle. Les sprites peuvent être créés dans de nombreux formats. Le sprite "traditionnel" est constitué d'images de pixels, importées dans GameMaker Studio 2 tant que *.png des dossiers. Cependant, vous pouvez également importer des sprites vectoriels enregistrés dans le *.swf format, ou des sprites d’ animation squelettique qui ont été sauvegardés en tant que *.json fichiers avec accompagnement *.atlas et *.png fichiers (pour plus d'informations sur ces types de fichiers image et sur leur importation dans GameMaker Studio 2, reportez-vous à la section Importation de sprites non bitmap ).

Les fonctions suivantes peuvent être utilisées pour obtenir des informations sur un actif d'image-objet. Veuillez noter que ces fonctions obtiennent les informations de l’image - objet de base (c’est-à-dire de son actif) et non de l’image-objet telle qu’elle est utilisée dans une instance. Par exemple, vous pouvez utiliser une variable d'instance sprite_width pour obtenir la largeur de l'image-objet qu'une instance utilise et cette largeur prendra en compte toute mise à l'échelle éventuellement appliquée. Cependant, si vous avez besoin de connaître la largeur d’un sprite normalement, sans aucune transformation, vous devez utiliser le paramètre sprite_get_width() fonction ci-dessous.
