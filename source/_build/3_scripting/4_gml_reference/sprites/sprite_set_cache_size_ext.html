sprite_set_cache_size_ext


Syntaxe:

sprite_set_cache_size_ext(ind, index, max);

Argument La description
ind L'index de l'image-objet pour modifier la taille du cache de.
index La sous-image de ind pour changer la taille du cache de.
max Le nombre maximal de copies en mémoire cache de l'image-objet pouvant être stockées.


Résultats

N/A


La description

Lorsque vous effectuez une fusion d'images, HTML5 ne peut pas le faire dynamiquement de la même manière qu'un exécutable peut être exécuté. Par conséquent, GameMaker Studio 2 doit enregistrer temporairement une copie mixte de l'image et la charger. Cette fonction définit le nombre de copies fusionnées de l'image-objet donnée pouvant être mises en cache avant que les anciennes ne soient écrasées. La valeur par défaut est 4. Ceci est appliqué à une seule sous-image donnée de l'image-objet.


Exemple:

sprite_set_cache_size_ext(sprite0, 0, 2);

Cela définira le cache d'image-objet de la première sous-image de sprite0 à 2 copies.
