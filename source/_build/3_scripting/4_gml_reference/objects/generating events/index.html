Générer des événements

GameMaker Studio 2 est entièrement piloté par les événements et toutes les actions sont le résultat de certains événements déclenchés dans un ordre spécifique ou d'une occurrence spécifique dans le jeu. Il y a un certain nombre d'événements différents, chacun avec une tâche ou une synchronisation spécifique et certains d'entre eux sont même divisés en "sous" des événements distincts. Normalement, ces événements sont exécutés par GameMaker Studio 2 lorsqu'il détecte un déclencheur dans le jeu ou chaque tick de jeu (comme c'est le cas avec les événements step et draw), mais vous pouvez également appliquer un événement à une instance dans la pièce actuelle. dans un morceau de code.

Pour plus d'informations sur les événements, veuillez consulter la section Événements.

Les fonctions suivantes concernent la génération d'événements:

  • event_inited
  • event_perform
  • event_perform_object
  • event_user

Il existe également un certain nombre de variables en lecture seule qui peuvent être utilisées avec les fonctions énumérées ci-dessus, particulièrement utiles pour le débogage:

  • event_action
  • event_number
  • event_object
  • type d'événement
