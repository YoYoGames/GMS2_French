application_surface_enable


Syntaxe:

application_surface_enable(enable);

Argument La description
enable Définir pour activer ( true ) ou désactiver ( false ) la surface d'application.


Résultats:

N/A


La description

Avec cette fonction, vous pouvez activer ou désactiver la surface de l'application. Par défaut, la surface d'application est activée et tous les dessins pour les événements Draw seront effectués ainsi que la surface dessinée à l'écran. Toutefois, sur certains appareils plus anciens et sur certains jeux de puces, cela peut nuire aux performances. Dans ces cas, vous pouvez désactiver le dessin sur la surface de l'application et tout dessin d'événement Draw sera alors effectué directement dans la mémoire tampon d'écran.

Vous pouvez utiliser cette fonction pour activer ou désactiver la surface de l'application à tout moment de votre jeu, mais vous devez savoir qu'il faut au moins une image (étape) pour que les modifications prennent effet. Le désactiver désactivera toutes les corrections de rapport d'aspect, comme défini dans les options de jeu pour la plate-forme cible. Cela signifie que vous devez avoir mis en place une méthode permettant d'adapter le jeu aux dimensions requises de l'appareil, de la fenêtre ou de l'écran. Si vous le rallumez, les options de jeu reprendront l’étape suivante.

Il est recommandé d'utiliser cette fonction avec précaution, car cela aura une incidence sur la manière dont tout est dessiné à l'écran. En général, vous voudriez l'avoir en tant qu'option dans un menu de votre jeu, ou ne l'appelez qu'une seule fois au début de votre jeu et écrivez votre code, que vous l'ayez activé ou désactivé.


Exemple:

if keyboard_check_pressed(vk_space)
   {
   if application_surface_is_enabled()
      {
      application_surface_enable(false);
      }
   else
      {
      application_surface_enable(true);
      }
   }

Le code ci-dessus vérifie si vous appuyez sur une touche et activez ou désactivez la surface d'application en fonction de son état (comme dans un menu d'options).
