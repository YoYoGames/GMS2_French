Dessiner et créer des primitives


Une primitive est un type de dessin réalisé à l'aide de points permettant de définir ce qui est affiché à l'écran. Ils peuvent être composés de points, de lignes ou de triangles simples. Ils peuvent être colorés, avoir un fondu alpha et même recevoir une texture, ce qui signifie qu'ils peuvent être utilisés pour créer des effets très excitants dans un jeu en 2D! Par exemple, vous pouvez créer un rectangle en utilisant plusieurs primitives de triangle, puis le texturer avec une image-objet indicateur et, dans l'événement de dessin d'une instance, placer ce drapeau "en vague" dans le vent et déformer l'image-objet que vous avez utilisée en modifiant simplement les points qui se forment. le primitif.

ATTENTION! Ces fonctions ne fonctionnent avec le module HTML5 que si vous avez activé OpenGL dans les options de jeu.

GameMaker Studio 2 vous permet de définir vos propres formats de vertex à partir desquels vous pouvez créer vos propres primitives personnalisées. Cela peut considérablement accélérer les opérations sur les shader ou peut être utilisé pour étendre leurs capacités et créer des effets surprenants. Vous trouverez des informations à ce sujet dans les sections suivantes:

  • vertex_format_begin
  • vertex_format_add_colour
  • vertex_format_add_position
  • vertex_format_add_position_3d
  • vertex_format_add_textcoord
  • vertex_format_add_normal
  • vertex_format_add_custom
  • vertex_format_end
  • vertex_format_delete
IMPORTANT! Une fois que vous avez créé votre format de sommet, l'ordre dans lequel vous avez défini les propriétés de sommet doit être respecté lors de la construction de vos primitives. Ainsi, si vous avez défini un format de sommet comme position, couleur, coordonnée de texture, vous devez ajouter ces propriétés à la primitive dans le même ordre, sinon vous obtiendrez une erreur.

IMPORTANT! Les primitives que vous construisez doivent suivre le format que vous avez défini à l'aide des fonctions de format de sommet. Ainsi, si vous avez défini un format de sommet avec des données de position uniquement, il est inutile de construire votre primitive avec des données de couleur. Notez que l'ordre dans lequel vous ajoutez des propriétés à la primitive que vous construisez est défini par l'ordre dans lequel vous avez ajouté ces propriétés lors de la création du format de sommet, donc si vous avez défini le format de vecteur avec la position, la couleur et la texture de l'ordre. coordonner, vous devez ajouter ces propriétés à la primitive construite dans le même ordre, sinon vous obtiendrez des erreurs.
