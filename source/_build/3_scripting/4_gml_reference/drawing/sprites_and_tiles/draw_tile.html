draw_tile

La description

Avec cette fonction, vous pouvez dessiner n'importe quelle tuile d'un ensemble de tuiles - rivaliser avec des rotations, des retournements et des miroirs - en définissant le cadre (s'il est animé) et la position dans la pièce. La valeur d'ID de groupe de mosaïques est l'index de groupe de mosaïques tel que défini dans l'EDI et les données de mosaïque sont les données "blob" contenant toutes les informations relatives à l'index de mosaïques et aux transformations appliquées. Vous pouvez utiliser la fonction tilemap_get pour obtenir les données de mosaïque qui peuvent ensuite être manipulées à l'aide des fonctions de mosaïque, ou vous pouvez les construire vous-même en utilisant les constantes de mosaïque suivantes avec la valeur d'index de la mosaïque à créer:

Constant La description
tile_rotate Utilisé pour définir / obtenir le bit de rotation d'un blob de données de mosaïque.
tile_mirror Utilisé pour définir / obtenir le bit miroir d'un blob de données de mosaïque.
tile_flip Utilisé pour définir / obtenir le bit de retournement d'un blob de données de mosaïque.
tile_index_mask Constante spéciale destinée à "et" avec le blob de données de mosaïque pour extraire l'index de mosaïque.


Vous pouvez utiliser "ou" au niveau du bit ces constantes avec la valeur d'index de mosaïque pour obtenir les données de mosaïque finales:

var tiledata = 8 | tile_rotate | tile_flip;


Le code ci-dessus crée simplement une variable locale pour stocker les données de mosaïque pour la mosaïque avec l'index de 8 sur le jeu de mosaïques, qui est ensuite pivoté et retourné. Cela peut ensuite être utilisé pour dessiner la mosaïque à cet index à partir du jeu de mosaïques que vous spécifiez. cela n'est lié à aucune couche de tilemap et la mosaïque sera dessinée à la profondeur de la couche de l'instance faisant le dessin, à la position x / y que vous spécifiez dans la pièce. Notez que vous pouvez également définir le cadre pour dessiner aussi. Si la mosaïque indexée fait partie d'un ensemble d'animation, vous pouvez définir le numéro de l'image ici (de 0 au nombre d'images - 1) et la mosaïque indexée sera simplement l'indicateur pour lequel la mosaïque animée que vous utilisez sera utilisée. être le cadre donné qui est dessiné. Si la tuile ne fait pas partie d'un ensemble animé, le cadre doit alors être défini sur 0.


Syntaxe:

draw_tile(tile set, tiledata, frame, x, y)


Arguments:

Argument La description
tile set L'ID du jeu de tuiles à utiliser
tiledata Les données de tuiles à utiliser
frame Le numéro d'image à utiliser par la mosaïque animée (0 par défaut pour les mosaïques non animées)
x La position x dans la pièce sur laquelle dessiner
y La position y dans la pièce sur laquelle dessiner


Résultats:

Real


Exemple:

var lay_id = layer_get_id("Tiles_sky");
var map_id = layer_tilemap_get_id(lay_id);
var data = tilemap_get(map_id, 0, 0);
tile_set_flip(data, true);
draw_tile(set_Wall, data, 0, x, y);

Le code ci-dessus va obtenir les données sur une tuile à partir du tilemap donné, puis positionner le drapeau "retourner" et utiliser ces données pour dessiner la tuile retournée à la position de l'instance.
