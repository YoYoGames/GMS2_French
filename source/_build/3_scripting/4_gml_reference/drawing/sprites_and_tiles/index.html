Dessin De Sprites Et De Tuiles

Même si vous pouvez définir des arrière-plans dans l'éditeur de salle et affecter des sprites à des instances, cela n'est souvent pas suffisamment flexible pour le jeu que vous créez. Par exemple, si une instance doit dessiner un menu, vous pouvez afficher plusieurs sprites de bouton à la fois, ou si vous avez un jeu avec des arrière-plans à parallaxe défilant, vous devrez pouvoir les définir par le code dans un fichier. instance de contrôleur d'un certain type. Dans ces situations, vous devez utiliser les fonctions suivantes qui permettent de dessiner des éléments d'image-objet et d'arrière-plan n'importe où dans la pièce. Certaines fonctions vous permettent également de manipuler la forme, la taille, la couleur et l'alpha de l'image-objet pour obtenir des effets spéciaux.

Vous pouvez affecter une image-objet à une instance. GameMaker Studio 2 dessine par défaut avec les propriétés données par les différentes variables de l'image-objet. Toutefois, si vous ajoutez un élément à l'événement draw, vous "prenez le contrôle" et vous devez informer GameMaker Studio 2 exacte. quoi dessiner. Cela signifie que si vous devez utiliser les fonctions suivantes pour définir exactement ce que l'instance doit dessiner et l'endroit où elle se trouve dans la pièce:

  • se dessiner
  • draw_sprite
  • draw_sprite_ext
  • draw_sprite_general
  • draw_sprite_part
  • draw_sprite_part_ext
  • draw_sprite_stretched
  • draw_sprite_stretched_ext
  • draw_sprite_pos
  • draw_sprite_tiled
  • draw_sprite_tiled_ext

Si vous utilisez des images-objets au format SWF, vous disposez également d'options supplémentaires pour contrôler l'anti-aliasing utilisé lors du dessin (voir Importation d'images-objets non bitmap pour plus d'informations sur ces types d'images-objets). Notez que ces fonctions fonctionnent au niveau global et définiront l'AA sur toutes les images-objets SWF vectorielles dessinées après leur appel.

  • draw_enable_swf_aa
  • draw_set_swf_aa_level
  • draw_get_swf_aa_level

Outre les fonctions répertoriées ci-dessus, il existe également deux fonctions spéciales conçues pour être utilisées avec des animations squelettiques (voir Importation d' images- objets non bitmap pour plus d'informations sur ces types d'images -objets et la section Animations squelettiques pour des fonctions spécifiques liées à ces types d'images-objets).:

  • draw_skeleton
  • draw_skeleton_instance
  • draw_skeleton_collision
  • draw_skeleton_time

Les mosaïques sont automatiquement dessinées en fonction des données qu'elles contiennent, du jeu de mosaïques utilisé et des propriétés de tilemap. Cependant, vous pouvez dessiner des mosaïques ou des cartes à l'aide des fonctions suivantes:

  • draw_tile
  • tilemap_draw
