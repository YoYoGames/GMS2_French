gpu_set_blendmode_ext


La description

Lorsque GameMaker Studio 2 dessine un pixel, il y a une couleur source (la couleur du pixel que nous allons dessiner) et une couleur de destination (la couleur qui se trouve déjà dans le pixel sur lequel nous sommes en train de dessiner). nouvelle couleur du pixel, les couleurs source et cible sont calculées en fonction du mode de fusion choisi. Chaque composant des couleurs est stocké sous forme de valeur à virgule flottante comprise entre 0 et 1, et la nouvelle couleur est calculée en multipliant chaque composant de la couleur source par un facteur et en multipliant chaque composant de la couleur de destination par un autre facteur, puis en ajoutant le paramètre souhaité. résultats ensemble composant par composant.

Cette fonction vous permet de définir les différents composants qui doivent être factorisés ensemble pour créer un mode de fusion personnalisé. La source et la destination ont toutes les deux une composante rouge, verte, bleue et alpha. Dans le graphique ci-dessous, les RGBA de la source sont pris en compte (Rs, Gs, Bs, As), tandis que les destinations sont (Rd, Gd, Bd, Ad). Les onze constantes disponibles peuvent être source ou destination (ou les deux) lorsqu'elles sont utilisées dans cette fonction.

Constant Facteur de mélange (rouge, vert, bleu, alpha)
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-As)
bm_src_alpha (As, As, As, As)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (Annonce, Annonce, Annonce, Annonce)
bm_inv_dest_alpha (1 annonce, 1 annonce, 1 annonce, 1 annonce)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) où f = min (As, 1-Ad)

Notez que vous pouvez soit fournir deux arguments individuels à cette fonction, soit un tableau d’arguments (tel qu’il est renvoyé par la fonction gpu_get_blendmode_ext par exemple). Si vous fournissez un tableau, il devrait contenir les deux éléments suivants:

  • [0] = mode de fusion source (la valeur par défaut est bm_src_alpha )
  • [1] = mode de fusion Destination (la valeur par défaut est bm_inv_src_alpha )

Il peut être assez difficile de visualiser le fonctionnement de ces différents modes de fusion. Vous trouverez ci-dessous un diagramme simple illustrant l'interaction de base d'un sprite dessiné avec différentes combinaisons de modes de fusion sur un même arrière-plan. Notez que les résultats réels du mode de fusion que vous utilisez dépendent beaucoup de la couleur de l'image-objet, de la valeur alpha de l'image-objet et de la couleur et de l'alpha de l'arrière-plan.

AVERTISSEMENT!: Tous les modes de fusion ne sont pas disponibles sur toutes les plateformes. Android, iOS et HTML5 sans WebGL activé ne pourra pas afficher correctement les modes suivants:

  • bm_src_colour
  • bm_inv_src_colour
  • bm_dest_colour
  • bm_inv_dest_colour
  • bm_src_alpha_sat


Syntaxe:

gpu_set_blendmode(src, dest);


Arguments:

Argument La description
src Facteur de mode de fusion source (voir les constantes ci-dessus).
dest Facteur de mode de fusion de destination (voir les constantes ci-dessus)


Résultats:

N/A


Exemple:

gpu_set_blendmode_ext(bm_src_alpha, bm_one);
draw_circle_colour(100, 100, 50, c_white, c_black, 0);
gpu_set_blendmode(bm_normal);

Cela transformera le noir en transparence, créant un effet de "rayonnement" du fait que le blanc est puissant à l'extérieur et s'affaiblit progressivement à partir du centre du cercle. Le mode de fusion est réinitialisé une fois le cercle dessiné afin de garantir que le mélange additif ne soit pas également appliqué à tout le reste.
