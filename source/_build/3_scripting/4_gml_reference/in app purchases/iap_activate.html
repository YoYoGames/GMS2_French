iap_activate

Permet de prendre en charge les achats intégrés.

Syntaxe:

iap_activate(product_index);


Argument La description
product_index L'index du ds_list qui contient les achats disponibles.


Retours: N / A


La description

Cette fonction permet de prendre en charge les achats GameMaker Studio 2 et prépare GameMaker Studio 2 en fournissant le produit. ds_list qui détient une série de ds_maps avec les informations sur chaque achat disponible. Cela signifie que vous devrez indiquer à GameMaker Studio 2 toutes les options d'achat disponibles en les configurant individuellement. ds_maps (un pour chaque achat disponible), qui sont ensuite stockés dans un ds_list, qui est ensuite "activé" avec cette fonction.

Les cartes d’achat individuelles qui doivent être stockées dans le ds_list devrait avoir le format suivant de paires clé-valeur:

  • " id " - L'ID du produit pour l'achat, par exemple: "LevelPack".

  • " title " - Le nom de l'achat, par exemple: "Level Pack 1".

  • " description " - La description de l'achat, par exemple: "Level Pack 1 pour Catch The Clown".

  • " prix " - Le prix de l'achat, par exemple: " $1 ".

Il est à noter que la seule clé essentielle pour toute configuration de magasin cible (par ID de produit) est "id".

L’activation des achats déclenchera également un événement IAP, qui crée un événement spécial. iap_data ds_map du type d'événement iap_ev_product. Ce ds_map aura la clé supplémentaire suivante:

  • " index " - L'ID du produit activé.

Si vous activez plusieurs produits, chaque produit déclenchera son propre événement IAP du type iap_ev_product où vous pouvez alors obtenir l'ID du produit. Il convient de noter que le magasin Google Play (pour Android ) ne peut traiter les détails de produits 20 qu'à la fois, ce qui peut entraîner des temps de chargement assez longs pour les applications comportant un nombre important de produits.



REMARQUE: Toutes les paires clé / valeur qui composent une carte d’achat sont des chaînes!


Exemple:

var purchaseList, purchase1;
purchaseList = ds_list_create();
purchase1 = ds_map_create();
ds_map_add(purchase1, "id", "LevelPack");
ds_map_add(purchase1, "title", "ExtraLevels1");
ds_map_add(purchase1, "description", "Level Pack 1 for Catch The Clown");
ds_map_add(purchase1, "price", "$1.00");
ds_list_add(purchaseList, purchase1);
iap_activate(purchaseList);
ds_map_destroy(purchase1);
ds_list_destroy(purchaseList);

Le code ci-dessus va créer un ds_list et un ds_map, qui est ensuite renseigné par les informations permettant d'effectuer un achat. Cette carte est ajoutée à la liste, puis activée comme achat disponible pour le jeu. Finalement, le ds_list et ds_map sont supprimés pour éviter les fuites de mémoire car ils ne sont plus nécessaires.
