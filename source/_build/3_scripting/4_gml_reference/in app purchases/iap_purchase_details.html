iap_purchase_details

Ajouter les détails d'achat d'un seul produit à un ds_map.

Syntaxe:

iap_purchase_details(product_id, ds_map);


Argument La description
product_id La chaîne d'ID de produit pour l'achat.
ds_map Le ds_map qui stockera les informations d'achat.


Retours: N / A


La description

Avec cette fonction, vous pouvez renseigner un répertoire déjà créé. ds_map avec un nombre de paires clé / valeur qui décrivent les détails d’achat du produit donné. Les données suivantes sont renvoyées dans la carte:

  • " produit " - La chaîne d'identification du produit.

  • " order " - L'ID de commande unique, le cas échéant (tous les magasins ne la prennent pas en charge).

  • " jeton " - Chaîne de jeton d'achat, le cas échéant (cette option n'est pas prise en charge par tous les magasins).

  • " charge utile " - La charge utile renvoyée par le magasin cible, le cas échéant (tous les achats ne l'exigent pas et tous les magasins ne le prennent pas en charge).

  • " receive " - La chaîne de réception, le cas échéant (toutes les boutiques ne la prennent pas en charge).

  • " réponse " - Valeur de réponse (sous forme de nombre réel), le cas échéant, basée sur les valeurs de réponse de Google Play (voir ici pour plus de détails).

La carte contiendra également une autre clé, " status ", qui peut être utilisée pour vérifier le statut actuel de l'achat. Cette clé aura l'une des constantes suivantes comme valeur:

Constant La description
iap_available Le produit est disponible ou le jeu attend le résultat d'un achat.
iap_failed La tentative d'achat a échoué.
iap_purchased Le produit a été acheté avec succès.
iap_canceled L'utilisateur a annulé l'achat.
iap_refunded L'achat a été annulé.



Les notes suivantes concernent les clés "jeton" et "reçu" de la carte renvoyée en référence au magasin cible utilisé:

  1. Pour Amazon Store, les entrées "jeton" et "reçu" d'achat sont traitées comme une seule et même chose.

  2. Pour Google Play, il n’existe pas de données de "reçu" et vous devez vérifier les achats via le "contenu" renvoyé (si vous effectuez une vérification via un contenu, il est recommandé de ne pas utiliser de valeurs codées en dur). GameMaker Studio 2 vérifiera toujours la signature de données renvoyée lors d'un achat par Google Play avec la clé publique fournie dans l'onglet IAP des Options de jeu.

  3. Pour Google Play, le "jeton" fourni avec un achat est destiné à être utilisé avec des achats consommateurs.

  4. Pour les magasins cibles iOS et MacOS, des données de "reçu" sont disponibles pour les achats, mais le "jeton" associé est généré en interne à des fins de développement.



NOTE: Le ds_map used n'est pas créé par la fonction, ce qui signifie que vous devez le créer précédemment et le supprimer de la mémoire lorsqu'il n'est pas utilisé à l'aide de la fonction appropriée.


Exemple:

var p_map = ds_map_create();
var p_index = ds_map_find_value(iap_data, "index");
iap_purchase_details(p_index, p_map);
if ds_map_find_value(p_map, "status") == iap_purchased
   {
   global.Gold += 100000;
   }
ds_map_destroy(p_map);

Le code ci-dessus crée un fichier ds_map, puis le remplit avec les paires clé / valeur pour l'achat avec l'ID de produit stocké dans la variable "p_index", que nous obtenons de la commande spéciale. iap_data ds_map créé exclusivement dans l' événement IAP. Si la touche "status" renvoie la constante iap_purchased, il va alors définir une variable globale.
