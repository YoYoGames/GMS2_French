os_is_paused

Vérifie si le système d'exploitation du périphérique est en pause ou non.

Syntaxe:

os_is_paused()


Retours: Booléen


La description

Lorsque vous développez pour des appareils mobiles, vous devez avoir un moyen de vérifier que l'appareil est toujours concentré sur votre jeu, en particulier si votre jeu est sur un appareil iOS, Android ou Windows Phone, il peut être interrompu par des appels téléphoniques et autres applications. Vous pouvez utiliser cette fonction pour vérifier si le périphérique est en pause ou non.

Il convient de noter que cette fonction exécute tout code contenu dans le bloc de code sur les périphériques, mais GameMaker Studio 2 est immédiatement suspendu lorsque le périphérique met en cache le jeu en tant que processus d'arrière-plan. Ainsi, si vous avez quelque chose comme l'exemple de code fourni ci-dessous, l'instance à créer n'apparaîtra ni n'exécutera aucun de ses événements jusqu'à ce que le jeu ait été redémarré à partir de l'état mis en cache. Cela signifie que vous préférerez peut-être avoir le os_is_paused la fonction appelle une alarme, puis dans l'événement d'alarme (qui sera défini lorsque le périphérique est en pause mais ne s'exécute pas tant qu'elle n'est pas interrompue), crée un menu de pause, redémarre la musique, etc.

Si vous utilisez cette fonction sur les plates-formes Windows, Ubuntu (Linux), MacOS ou HTML5, la fonction n'est activée que pour une étape. true quand le jeu perd le focus, puis il reviendra false à nouveau jusqu'à ce que le jeu revienne à se concentrer (et le perd à nouveau). En effet, ces plates-formes ne mettent pas en cache les applications et les jeux. Celles-ci continuent donc de s'exécuter en arrière-plan.

REMARQUE: sur les appareils mobiles, les sons et la musique sont arrêtés lorsque le système d'exploitation est mis en pause. Ils ne redémarreront pas à moins que vous ne les lanciez spécifiquement avec la fonction appropriée.


Exemple:

if os_is_paused()
   {
   if !instance_exists(obj_PauseMenu) instance_create_layer(0, 0, "Controllers", obj_PauseMenu);
   }

Le code ci-dessus vérifie si le jeu a été suspendu et, le cas échéant, il crée une instance de l'objet "obj_PauseMenu" s'il n'en existe pas déjà.
