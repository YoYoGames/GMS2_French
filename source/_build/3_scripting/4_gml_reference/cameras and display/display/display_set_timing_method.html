display_set_timing_method


La description

Cette fonction peut être utilisée pour définir la méthode de synchronisation à utiliser pour le rendu de votre partie. La méthode peut être l'une des constantes suivantes:

Constant La description
tm_sleep Utilisez la valeur de la marge de sommeil comme méthode principale de synchronisation
tm_countvsyncs Utilisez la synchronisation vsync comme méthode de synchronisation principale (par défaut pour toutes les plates-formes prises en charge)


La méthode de synchronisation vsync utilise le support des plates-formes cibles pour la synchronisation verticale afin de fournir un point d'ancrage pour les calculs de rendu des jeux. Le réglage du mode sur la marge de veille tente simplement de garantir que chaque image dure le temps correct (disons 1 / 30ème). ou 1 / 60ème de seconde) en attendant ou en dormant. En général, le minutage vsync par défaut donnera les résultats les plus lisses, mais notez que lorsque vous utilisez la méthode vsync, les valeurs de la marge de veille restent pertinentes, même si leur impact est réduit. Nous vous recommandons de conserver cette valeur à sa valeur par défaut.

Par défaut, sur toutes les plates-formes sauf PS4, Ubuntu et HTML5, GameMaker Studio 2 utilisera la méthode de synchronisation vsync, tandis que sur les plates-formes non prises en charge, seule la synchronisation de la marge de veille peut être utilisée.

Si vous souhaitez définir la marge de sommeil, vous pouvez le faire en utilisant la fonction display_set_sleep_margin() et vous pouvez trouver quelle méthode de synchronisation est actuellement utilisée avec la fonction display_get_timing_method().


Syntaxe:

display_set_timing_method(method);

Argument La description
method La méthode de synchronisation à utiliser (voir la liste des constantes ci-dessus)


Résultats:

N/A


Exemple:

if display_get_timing_method() != tm_sleep
{
display_set_timing_method(tm_sleep);
if display_get_sleep_margin() != 20
    {
    display_set_sleep_margin(20);
    }
}

Le code ci-dessus vérifiera la méthode de synchronisation, puis s’il n’est pas défini sur tm_sleep il sera réglé sur cela et la limite de sommeil réglée sur 20.
