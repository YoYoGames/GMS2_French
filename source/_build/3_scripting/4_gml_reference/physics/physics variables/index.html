Variables de physique

Comme pour les collisions et le système de mouvement «traditionnels», le système de physique a également une série de variables d'instance affectées à chaque instance d'un objet lors de sa création. Certaines de ces propriétés peuvent être définies par l'utilisateur dans GML, mais beaucoup sont en lecture seule et d'autres ne sont applicables que dans l'événement de collision.

Les variables suivantes existent dans tous les objets activés de la physique et peuvent être lues et écrites (ie: changées) en utilisant le code des instances et des scripts:

  • phy_active
  • phy_angular_velocity
  • phy_angular_damping
  • phy_linear_velocity_x
  • phy_linear_velocity_y
  • phy_linear_damping
  • phy_speed_x
  • phy_speed_y
  • phy_position_x
  • phy_position_y
  • phy_position_xprevious
  • phy_position_yprevious
  • phy_rotation
  • phy_fixed_rotation
  • phy_bullet

Certaines variables sont variables de lecture, ce qui signifie que les informations qu'ils contiennent ne peut pas être changé par code instance ou des scripts (mais il peut être modifié en interne par GameMaker Studio 2 qui est la raison pour laquelle ceux - ci sont variables et non constantes). Le tableau suivant montre les variables en lecture seule qui existent dans tous les objets activés pour la physique:

  • phy_speed
  • phy_com_x
  • phy_com_y
  • phy_dynamic
  • phy_kinematic
  • phy_inertia
  • phy_mass
  • phy_sleeping

Enfin, un certain nombre de variables temporaires existent dans les objets activés de physique et sont créées uniquement dans l'événement de collision. Ces variables sont similaires à celles listées ci-dessus et sont en lecture seule:

  • phy_collision_points
  • phy_collision_x
  • phy_collision_y
  • phy_col_normal_x
  • phy_col_normal_y
