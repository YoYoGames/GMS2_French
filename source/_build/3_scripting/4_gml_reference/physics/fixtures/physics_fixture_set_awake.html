physics_fixture_set_awake

Cette fonction indique à GameMaker Studio 2 que le projecteur doit être traité comme éveillé ou non.

Syntaxe:

physics_fixture_set_awake(fixture, state)


Argument La description
fixture l'index de l'appareil
flag si un luminaire est éveillé ( true ) ou pas ( false )


Retours: N / A


La description

Lorsque vous démarrez une pièce avec la physique, ou liez des fixtures à des instances, les appareils sont toujours considérés comme "éveillés"... c'est-à-dire qu'ils traitent des événements et interagissent avec les instances environnantes. Cependant, cela peut parfois conduire à des problèmes, en particulier si vous avez un certain nombre d'instances avec des appareils qui sont côte à côte lorsqu'une pièce démarre (pensez à une tour composée de plusieurs instances avec des appareils) car elles sont "éveillées". peut-être changer de position en raison du début soudain de la gravité et des collisions qui les affectent. Dans ce cas, vous pouvez utiliser cette fonction pour marquer l'appareil comme étant "endormi", auquel cas il ne traitera rien lors de sa création dans la pièce.


Exemple:

physics_fixture_set_awake(fix_Cloud, true);

Le code ci-dessus signale que l'appareil est "éveillé" lorsqu'il est lié à une instance.
