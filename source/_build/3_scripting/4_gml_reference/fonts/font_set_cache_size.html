font_set_cache_size


La description

Lors de la fusion et de la coloration d'images, HTML5 ne peut pas le faire dynamiquement de la même manière qu'un exécutable peut être exécuté. Par conséquent, GameMaker Studio 2 doit enregistrer temporairement une copie mixte des images et les charger en cas de besoin. Cette fonction définit combien de copies fusionnées de la police donnée peuvent être mises en cache avant que les anciennes ne soient écrasées. La valeur par défaut est 4.

Remarque: pour les polices sprite, vous devez utiliser la fonction équivalente pour les sprites, sprite_set_cache_size.


Syntaxe:

font_set_cache_size(ind, max);


Argument La description
ind L'index de la police pour changer la taille du cache de.
max Le nombre maximal de copies en mémoire cache de la police pouvant être stockée.


Résultats:

N/A


Exemple:

font_set_cache_size(fnt_MainMenu, 2);

Cela va mettre le cache de police de la police indexée dans la variable "fnt_MainMenu" à 2 copies.
