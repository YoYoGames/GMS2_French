La mise en réseau

Les fonctions réseau de GameMaker Studio 2 sont basées sur des sockets, qui sont un standard de l’industrie sur toutes les plateformes. Avec les sockets, vous avez la possibilité de créer des clients et des serveurs en utilisant un simple GML, ce qui vous permet de créer même un jeu à un seul joueur en utilisant le modèle client / serveur (ce qui permettra d’ajouter facilement un mode multijoueur ultérieurement).

Fondamentalement, une socket est un objet qui peut envoyer, recevoir, connecter et écouter des ports sur le réseau. Ces fonctions concernent uniquement les connexions TCP / IP, sur lesquelles Internet est basé (IPv4 pour être plus précis). En substance, cela vous permet de connecter un socket en utilisant une adresse IP (disons 192.168.1.10) à un autre socket avec une autre adresse IP (par exemple 192.168.1.11). Une fois connectés, ces deux sockets peuvent échanger des données en utilisant le network_send_* fonctions (comme network_send_packet ) et l’événement asynchrone dédié qu’ils déclenchent. Cet événement asynchrone générera une ds_map contenant les données reçues ainsi que d'autres détails. Vous trouverez les détails complets dans la section relative à l'événement asynchrone réseau.

Au lieu de programmes qui doivent lire et gérer chaque bit de trafic réseau entrant dans une machine, les adresses IP traitent également des ports, numérotés de 0 à 65535. Cela permet à chaque programme d’obtenir uniquement les paquets de données qui l’intéressent. dans, et pas tout ce qui a été envoyé par chaque programme. Cela économise également un temps processeur considérable, car il réduit considérablement le traitement des données. Ainsi, avec GameMaker Studio 2 vous pouvez également "lister" les ports. Ainsi, un serveur créera un socket, puis le liera à un port spécifique à écouter. Il sera alors informé des connexions, des déconnexions et des transferts de données à partir de ce socket.

Les "paquets" de données sont eux-mêmes constitués de données binaires extraites d'un tampon. Ce tampon peut être créé et manipulé à l'aide des fonctions Tampon de GameMaker Studio 2.

REMARQUE: ces fonctions ne sont actuellement pas disponibles pour le module cible HTML5.

Vous pouvez utiliser les fonctions suivantes pour la mise en réseau:

  • serveur_créé_réseau
  • network_create_server_raw
  • network_create_socket
  • network_create_socket_ext
  • réseau_connexion
  • network_connect_raw
  • network_resolve
  • network_set_config
  • network_set_timeout
  • network_send_broadcast
  • network_send_packet
  • network_send_raw
  • network_send_udp
  • network_send_udp_raw
  • network_destroy
