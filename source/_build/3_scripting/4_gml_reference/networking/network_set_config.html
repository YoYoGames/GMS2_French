network_set_config

Syntaxe:

network_set_config(config_value, setting);


Argument La description
config_value La constante de valeur de configuration à définir (listée ci-dessous).
setting Le réglage de la valeur de configuration.


Retours: String


La description

Avec cette fonction, vous pouvez définir différentes configurations de réseau. Ces configurations sont données sous forme de constantes et le réglage dépendra de la constante que vous avez choisie. Le tableau ci-dessous répertorie les constantes disponibles et leurs paramètres:

Constant La description Réglage
network_config_connect_timeout Définir une valeur de délai de connexion A value in milliseconds
network_config_use_non_blocking_socket Dites à GameMaker Studio 2 ne pas bloquer lors de la connexion. 0 = block, 1 = don't block
network_config_enable_reliable_udp Active le protocole "fiable UDP" pour un socket UDP existant A value in milliseconds
network_config_disable_reliable_udp Désactive le protocole "fiable UDP" pour un fichier UDP existant. 0 = block, 1 = don't block



La première option ( network_config_connect_timeout ) définit simplement le délai d'expiration de la connexion à un serveur, mais ne modifie pas le comportement de la connexion en dehors du temps d'attente. Il est recommandé de ne pas régler ce paramètre trop bas et de le maintenir à environ 1 000 pour un jeu en réseau uniquement ou à environ 4 000 pour Internet, sinon le jeu pourrait ne pas se connecter au hasard. Si vous souhaitez définir une valeur de délai d'attente pour l'envoi / la réception de paquets, utilisez la fonction network_set_timeout().

La deuxième option ( network_config_use_non_blocking_socket ) signifie que les fonctions de connexion au réseau renverront toutes un socket_id instantanément, mais vous ne pouvez ni envoyer ni recevoir sur ce socket tant que vous n'avez pas reçu un événement réseau asynchrone. L’événement déclenché aura la clé "type" réglée sur network_type_non_blocking_connect (vous trouverez plus de détails sur la page relative à l' événement de mise en réseau asynchrone ). Il s'agit d'un paramètre global, car GameMaker Studio 2 ne prend pas en charge une GameMaker Studio 2 de blocage et de non-blocage dans une application.

Les troisième et quatrième options ( network_config_enable_reliable_udp, network_config_disable_reliable_udp ) permettent d'activer ou de désactiver le protocole UDP fiable de GameMaker Studio 2 et ne doivent donc être utilisés qu'avec des connexions UDP. Ce qui est "fiable" dans ce cas, cela signifie que, lorsqu'il est activé aux deux extrémités d'une connexion, GameMaker Studio 2 tentera de vérifier si vos paquets sont bien arrivés et de renvoyer ceux qui n'arrivent pas (remarque, nous ne garantissons pas dans vos paquets arriveront dans l’ordre de transmission). Si vous utilisez un type de socket fiable, un en-tête de 12 octets sera ajouté à tous les paquets udp. Il contiendra les informations nécessaires à GameMaker Studio 2 pour rechercher d' GameMaker Studio 2 erreurs dans les paquets et pour renvoyer les paquets manquants.


Exemple:

network_set_config(network_config_connect_timeout, 1000);
network_set_config(network_config_use_non_blocking_socket, 1);

Le code ci-dessus définira le délai d'expiration de la connexion réseau sur GameMaker Studio 2 et indiquera à GameMaker Studio 2 ne pas bloquer la connexion.
