Fichiers texte

Les fichiers texte sont un moyen utile de stocker de grandes quantités de données à l'extérieur de votre jeu. Contrairement aux fichiers ini, vous pouvez avoir jusqu'à 32 fichiers texte ouverts en même temps, et vous pouvez aussi lire et écrire de gros morceaux de données. Cependant, c'est à vous de décider comment ces données doivent être structurées car il n'y a pas de structure "section" ou "clé" comme avec *.ini fichiers

S'il vous plaît noter que pour les jeux avec des localisations, il est très important que les lettres accentuées peuvent être lues à partir de fichiers externes. Cela signifie que vous devez créer le fichier txt au format UTF8 d' abord, puis l' ajouter dans GameMaker Studio 2 en tant que fichier inclus de sorte qu'il est exporté sur l' exécution du jeu et utilisé à la place du fichier format par défaut ANSI txt qui est créé par le GameMaker Studio 2 fichier GameMaker Studio 2 fonctionne quand aucun fichier n'est fourni précédemment. De cette façon, vous pouvez lire et écrire correctement avec tous les accents et les lettres non-romaines étant maintenues.

Les fonctions suivantes concernent les fichiers:

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
ATTENTION! Cette fonction peut ne pas fonctionner comme prévu car GameMaker Studio 2 est en sandbox! Veuillez consulter la section sur les limites du système de fichiers pour plus d'informations.
