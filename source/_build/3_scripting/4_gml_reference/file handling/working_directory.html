directeur de travail


La description

Working_directory peut en fait renvoyer deux valeurs différentes selon l’utilisation que vous en faites. Si vous écrivez un fichier sur le disque, working_directory pointe vers la zone de stockage local réservée à votre jeu sur le périphérique cible (sous Windows, il s'agit de "% LOCALAPPDATA% / gamedir /" où "gamedir" est le répertoire avec le nom de votre jeu). Cependant, lors de la lecture du disque, working_directory peut être le stockage local ou la zone où les fichiers inclus sont stockés.

NOTE: Le working_directory variable renverra le chemin, y compris la dernière barre oblique inverse.

Par exemple, si vous avez un défaut *.txt fichier inclus avec votre jeu et lisez-le en mémoire en utilisant working_directory, ce fichier sera alors extrait de la zone où les fichiers inclus sont stockés. Si vous choisissez ensuite d'écrire ces informations dans un fichier texte à l'aide de working_directory, le fichier sera écrit sur le stockage local de la plate-forme cible, où il sera lu à partir de la prochaine fois, à moins que vous ne GameMaker Studio 2 de supprimer le fichier. Dans ce cas, working_directory Une fois de plus, pointez sur la zone où sont stockés les fichiers inclus.

ATTENTION! Cette fonction risque de ne pas fonctionner GameMaker Studio 2 car GameMaker Studio 2 mis en bac à sable! Veuillez consulter la section sur les limites du système de fichiers pour plus d'informations.


Syntaxe:

working_directory


Résultats:

String


Exemple:

ini_open(working_directory + "temp_ini.ini");

Cela ouvrira un fichier ini à partir du répertoire de travail du jeu (en le créant s'il n'existe pas déjà). Il peut s'agir du stockage local ou de la zone où se trouvent les fichiers inclus, selon que le fichier recherché existe ou non.
