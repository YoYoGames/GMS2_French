json_encode


La description

JSON (JavaScript Object Notation) est un format léger d’échange de données, facile à lire et à écrire, pour les utilisateurs comme pour les ordinateurs. Il est construit sur deux structures de base:

  • Une collection de paires nom / valeur, appelée un ds_map dans GameMaker Studio 2 mais également appelé "dictionnaire" ou "objet".

  • Une liste ordonnée de valeurs, appelée un ds_list dans GameMaker Studio 2 mais cela peut aussi être appelé "tableau" ou "séquence".

json_encode prend un ds_map que vous avez précédemment créé et codé en tant que chaîne JSON que vous pouvez ensuite utiliser comme (par exemple) une partie d'un http_post_string() appel, ou alors il peut être stocké en externe, écrit dans un fichier.

REMARQUE: La fonctionnalité hiérarchique de JSON est disponible via les fonctions spéciales ds_map et ds_list, vous permettant ainsi de coder des sous-listes et des mappes.


Syntaxe:

json_encode(map)


Argument La description
map un ds_map avec les informations à encoder


Résultats:

string


Exemple:

var hiscore_map, i, str;
hiscore_map = ds_map_create();
for (i = 0; i < 10; i ++;)
   {
   ds_map_add(name[i], score[i]);
   }
str = json_encode(hiscore_map);
get[0] = http_post_string("http://www.angusgames.com/game?game_id=" + string(global.game_id), str)
ds_map_destroy(hiscore_map);

Le code ci-dessus crée un fichier ds_map, puis parcourt les tableaux de noms et de scores, ajoutant chaque paire clé / valeur à la nouvelle carte. Ensuite, cette carte est encodée en utilisant json_encode et stocké sous forme de chaîne dans la variable "str". Cette chaîne est ensuite envoyée à un serveur Web à l'aide de http_post_string et ds_map est détruit pour éviter une fuite de mémoire car elle n’est plus nécessaire.
