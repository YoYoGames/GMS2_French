file_find_first

Syntaxe:

file_find_first(mask, attr);


Argument La description
mask Le masque à utiliser pour la recherche.
attr L'attribut de fichier spécifique à rechercher.


Retourne: Chaîne


La description

Cette fonction renverra le nom du premier fichier qui satisfait le masque et les attributs. Si aucun fichier n'existe, une chaîne vide est renvoyée. Le masque peut contenir un chemin et peut contenir des wildchars, par exemple 'C: \ temp \ *. Doc'. Les attributs donnent les fichiers supplémentaires que vous voulez voir, donc les fichiers normaux sont toujours retournés quand ils satisfont le masque. Vous pouvez ajouter les constantes suivantes pour voir le type de fichiers que vous voulez (si vous ne souhaitez pas ajouter d'attributs, utilisez 0):

  • fa_readonly - Fichiers en lecture seule
  • fa_hidden - fichiers cachés
  • fa_sysfile - fichiers système
  • fa_volumeid - fichiers volume-id
  • fa_directory - répertoires
  • fa_archive - fichiers archivés


ATTENTION! Cette fonction peut ne pas fonctionner comme prévu car GameMaker Studio 2 est en sandbox! Veuillez consulter la section sur les limites du système de fichiers pour plus d'informations.

REMARQUE: cette fonction ne fonctionnera pas du tout sur les plates-formes cible JS (Windows8, Tizen et HTML5) et les indicateurs de filtre ne fonctionneront que sur Windows et WinPhone (utilisez 0 pour toutes les autres plates-formes).


Exemple:

if directory_exists("\User Content")
   {
   file = file_find_first("\User Content\*.doc", fa_readonly);
   }

Cela vérifiera pour voir si le répertoire spécifié existe alors, si c'est le cas, allez là et renvoyez le premier "lecture seule" doc fichier trouvé.
