La gestion des fichiers


À un moment donné, lors de la création de votre jeu, il sera nécessaire de stocker des informations en externe et de pouvoir les récupérer. Par exemple, vous souhaiterez peut-être stocker les informations de connexion, les scores, ou peut-être la progression du joueur dans vos niveaux. Quelle que soit la raison, vous devrez savoir utiliser les différentes fonctions de fichier disponibles lors de l' GameMaker Studio 2 de GameMaker Studio 2.

Il est très important de réaliser que GameMaker Studio 2 est un Sandbox, ce qui signifie qu’il existe un certain nombre de limites et de complications mineures lors de l’utilisation des fonctions de système de fichiers disponibles. Il est également intéressant de noter que le système de fichiers fonctionnera légèrement différemment en fonction des fonctions. la plate-forme que vous ciblez. Pour plus d'informations, consultez la section suivante:

  1. Le système de fichiers

Si Steam vous a accordé l'autorisation d'utiliser son API dédiée avec votre jeu, vous constaterez qu'un nouvel ensemble de fonctions de fichiers est également disponible. Celles-ci sont expliquées dans la section API Steam de ce manuel.

Enfin, vous devez également noter qu’il existe certaines différences distinctes entre certaines plates-formes cibles qui doivent être prises en compte lors de l’utilisation des fonctions de fichier. Ceux-ci sont:

  • iOS: sur iOS, si vous chargez un fichier inclus à partir d'un dossier, c.-à-d. avec un chemin similaire à " Backgrounds/background1.png ", vous n'avez PAS besoin d'inclure le dossier dans le chemin et il vous suffit de fournir le nom du fichier. Sur toutes les autres plates-formes, vous devez fournir le (s) dossier (s) dans le chemin du fichier.

  • macOS et Ubuntu (Linux): sur macOS et Ubunutu, les noms de fichiers et les chemins d'accès sont sensibles à la casse sur ces plates-formes. Par conséquent, pour éviter tout problème potentiel, GameMaker Studio 2 fera automatiquement utiliser les minuscules par défaut.

  • HTML5: sous HTML5, l'enregistrement et le chargement depuis et vers le stockage local fonctionnent comme indiqué dans les différentes pages de fonctions. Toutefois, si vous chargez un fichier à partir d'un serveur, vous devez uniquement utiliser la fonction. buffer_load_async() car le chargement synchrone est obsolète sur la plupart des navigateurs et sera éventuellement rendu obsolète.

Il est évident que vous aurez parfois besoin de référencer le répertoire dans lequel les fichiers sont GameMaker Studio 2 et que GameMaker Studio 2 dispose également de fonctions spéciales relatives aux différents répertoires possibles dont vous pourriez avoir besoin:

  • répertoire_exists
  • répertoire_créer
  • destruction_répertoire
  • répertoire_temp
  • directeur de travail
  • répertoire_programme

Lorsqu'il s'agit de fichiers externes, il est toujours possible que l'utilisateur final puisse ouvrir et modifier les informations qu'il contient, et ainsi changer votre jeu. Cela peut entraîner des éléments de jeu cassés ou des scores frauduleux en ligne (par exemple). GameMaker Studio 2 vous propose donc certaines fonctions d'encodage de base, ainsi que des fonctions permettant d'effectuer des contrôles md5 sur les chaînes et les fichiers afin de vous assurer qu'ils ont maintenu leur intégrité avant d'être utilisé. Deux fonctions sont également fournies pour l’encodage et le décodage de chaînes de format JSON (JavaScript Object Notation).

  • base64_encode
  • base64_decode
  • json_encode
  • json_decode
  • md5_string_utf8
  • md5_string_unicode
  • md5_file
  • sha1_string_utf8
  • sha1_string_unicode
  • sha1_file
  • zip_unzip
  • load_csv
REMARQUE: l' encodage n'est PAS un cryptage! Un encodage en base64 rend le fichier illisible à l'oeil nu et nécessitera un effort de la part de l'utilisateur pour le décoder, mais il n'est pas sécurisé contre le piratage. Il est recommandé de mélanger ces fonctions avec votre propre cryptage (il existe de nombreuses formes de cryptage et des scripts sont disponibles sur Internet).

IMPORTANT! Tous les projets GameMaker Studio 2 ont un fichier appelé " opions.ini ", vous ne pouvez donc nommer aucun fichier portant ce nom (vous obtiendrez un échec de l 'identifiant utilisateur lorsque vous tenterez de compiler votre projet de jeu).
