ini_open


La description

Ceci ouvre un fichier ini_file pour la lecture et / / l’écriture. Si le fichier ini_file n'existe pas à l'emplacement que vous vérifiez, GameMaker Studio 2 peut en créer un, mais uniquement si vous y écrivez des données. Si vous avez uniquement lu des informations à partir du fichier ini, les valeurs par défaut de la fonction de lecture seront renvoyées, mais le fichier ini ne sera pas créé.

Veuillez noter que vous ne pouvez ouvrir qu’un seul fichier ini à la fois et n’oubliez pas d’utiliser ini_close une fois que vous avez terminé la lecture / écriture à partir du fichier.ini car les informations ne sont pas réellement stockées sur le disque avant cette date (elles sont également stockées en mémoire jusqu'à la fermeture du fichier).

ATTENTION! Cette fonction risque de ne pas fonctionner GameMaker Studio 2 car GameMaker Studio 2 mis en bac à sable! Veuillez consulter la section sur les limites du système de fichiers pour plus d'informations.


Syntaxe:

ini_open(name);


Argument La description
name Le nom de fichier pour le fichier.ini.


Résultats:

N/A


Exemple:

ini_open("Settings/savedata.ini");
score = ini_read_real("save1", "score", 0);
ini_close();

Ceci ouvrira 'savedata.ini' et lira la valeur du score sous la section "save1" avec la clé "score" dedans, puis refermera le fichier.ini. S'il n'y a pas de valeur sous "save1", "score" ou s'il n'y a pas de fichier "savedata.ini", le score sera défini sur 0 (valeur par défaut). Notez que le fichier ini a été placé dans le sous-répertoire "Paramètres", qui est le dossier qui contient le fichier INI dans les fichiers inclus dans l'arborescence des ressources.
