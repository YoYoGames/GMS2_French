Fichiers Ini


Les fichiers INI sont de petits fichiers légers compatibles avec la plupart des plateformes. Elles sont idéales pour stocker de petites informations, telles que les préférences d'interface, les meilleurs scores locaux, les données de niveau, etc., et constituent donc un moyen simple et efficace pour commencer à enregistrer des informations sur vos jeux. Toutefois, il convient de noter qu’ils ont une limite sur les données qu’ils peuvent stocker, ce qui est un maximum de 64 Ko. Par conséquent, si vous enregistrez des éléments tels que des structures de données, vous voudrez peut-être examiner certaines des autres fonctions de fichier telles que Texte. Fichiers.

Veuillez noter que pour les jeux avec des localisations, il est très important que les lettres accentuées puissent être lues à partir de fichiers externes. Cela signifie que vous devez créer le fichier ini au format UTF8 d' abord, puis l' ajouter dans GameMaker Studio 2 en tant que fichier inclus de sorte qu'il est exporté sur l' exécution du jeu et utilisé à la place du format par défaut ANSI fichier ini qui est créé par l'ini fonctionne lorsque aucun n’a déjà été fourni. De cette façon, vous pouvez lire et écrire correctement en maintenant tous les accents et les lettres non romaines.

Les fonctions suivantes existent pour les fichiers ini:

  • ini_open
  • ini_close
  • ini_write_real
  • ini_write_string
  • ini_read_real
  • ini_read_string
  • ini_key_exists
  • ini_section_exists
  • ini_key_delete
  • ini_section_delete
  • ini_open_from_string
ATTENTION! Ces fonctions risquent de ne pas fonctionner GameMaker Studio 2 car GameMaker Studio 2 mis en bac à sable! Veuillez consulter la section sur les limites du système de fichiers pour plus d'informations.
