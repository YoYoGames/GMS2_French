clickable_add_ext

Syntaxe:

clickable_add_ext(x, y, tpe, url, target, params, scale, alpha);


Argument La description
x La position x dans la fenêtre.
y La position y dans la fenêtre.
tpe L’entrée de la page de texture du sprite à utiliser (utilisez sprite_get_tpe ).
url L'URL (adresse du site Web) vers laquelle effectuer le lien, ou le script à exécuter.
target C'est la zone cible pour ouvrir l'URL (voir description).
params Les différents paramètres de l'icône (voir description).
scale L'échelle de l'icône (1 par défaut).
alpha L'image alpha de l'icône (1 par défaut).


Retours: réel


La description

Avec cette fonction, vous pouvez créer une icône personnalisée, cliquable, qui agit comme un "bouton" en dehors du canevas du jeu, mais dans la fenêtre même du navigateur. La fonction retournera une valeur id qui doit être stockée dans une variable à utiliser dans toutes les fonctions de bouton ultérieures qui font référence à la nouvelle icône. Cette fonction est très utile pour empêcher le navigateur de créer une fenêtre contextuelle lorsque vous cliquez sur le bouton, car cela se fait tout au long de GameMaker Studio 2.

Cette fonction fonctionne exactement comme le clickable_add function, où vous devez définir la position par rapport au (0,0) de la fenêtre du navigateur, indiquez l’entrée de la page de texture pour l’image-objet que vous souhaitez attribuer à l’icône du bouton (voir: sprite_get_tpe ), spécifiez soit une URL à ouvrir, soit un script (de l’arborescence de ressources de GameMaker Studio 2 ) à ouvrir ou à exécuter, puis définissez la cible et les paramètres à utiliser. Cependant, cette fonction comporte également deux options supplémentaires vous permettant de définir l'icône alpha de 0 (totalement transparent) à 1 (totalement opaque) et l'échelle de l'icône, qui est une valeur de multiplicateur appliquée à l'image-objet utilisée. Ainsi, une valeur de 1 est l'échelle par défaut, tandis que (par exemple) 0,5 serait une demi-échelle et 2 serait le double.

Il convient également de noter que si l'argument d'URL que vous indiquez est le nom d'un script (c'est-à-dire qu'il ne commence pas par "http: //") ET que le script s'appelle gmcallback_* alors vous pouvez obtenir des flux directs dans un événement html5. GameMaker Studio 2 noms de fonction de GameMaker Studio 2 ne sont pas obscurcis s’ils commencent par gmcallback_.


Exemple:

home_but = clickable_add_ext(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0", 2, 1);

Le code ci-dessus crée une icône DOM cliquable à la position (32, 32) de la page sur laquelle le canevas du jeu est exécuté. L'icône utilise l'image-objet référencée dans la page de texture en tant que "spr_MS_Home" et, lorsque vous cliquez dessus, l'icône ouvrira une nouvelle fenêtre pour l'URL spécifiée et avec les propriétés définies pour cette fenêtre. le bouton sera également redimensionné à deux fois la taille de l'image-objet d'origine et à une demi-image alpha de 1.
