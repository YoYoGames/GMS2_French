clickable_add


Syntaxe:

clickable_add(x, y, tpe, url, target, params);


Argument La description
x La position x dans la fenêtre.
y La position y dans la fenêtre.
tpe L’entrée de la page de texture du sprite à utiliser (utilisez sprite_get_tpe ).
url L'URL (adresse du site Web) vers laquelle effectuer le lien, ou le script à exécuter.
target C'est la zone cible pour ouvrir l'URL (voir description).
params Les différents paramètres de l'icône (voir description).


Retours: réel


La description

Avec cette fonction, vous pouvez créer une icône personnalisée, cliquable, qui agit comme un "bouton" en dehors du canevas du jeu, mais dans la fenêtre même du navigateur. La fonction retournera une valeur id qui doit être stockée dans une variable à utiliser dans toutes les fonctions de bouton ultérieures qui font référence à la nouvelle icône. Cette fonction est très utile pour empêcher le navigateur de créer une fenêtre contextuelle lorsque vous cliquez sur le bouton, car cela se fait tout au long de GameMaker Studio 2.

Vous devez d’abord définir la position par rapport au (0,0) de la fenêtre du navigateur (coin supérieur gauche), puis donner l’entrée de la page de texture pour l’image-objet que vous souhaitez attribuer à l’icône du bouton (voir: sprite_get_tpe ). Vous pouvez ensuite spécifier soit une URL à ouvrir, soit un script (à partir de l'arborescence de ressources de GameMaker Studio 2 ) à ouvrir ou à exécuter. Le paramètre "cible" que vous spécifiez est identique à la valeur "nom" standard de JavaScript lorsque vous utilisez la commande open() method (sachez que tout sauf _self peut entraîner le blocage du navigateur ou demander à l'utilisateur s'il le permet) et l'argument "params" est identique au paramètre JavaScript "specs" permettant de contrôler les propriétés de la nouvelle La fenêtre / l’onglet devrait s’afficher (tous les navigateurs ne peuvent pas supporter toutes les fonctionnalités). Si vous utilisez l'icône pour exécuter un script interne, vous pouvez laisser ces deux derniers arguments sous forme de chaînes vides "".

Les cibles valides sont:

Cible La description
_blanc Ouvre le document lié dans une nouvelle fenêtre ou un nouvel onglet.
_soi Ouvre le document lié dans le même cadre que celui sur lequel il a été cliqué (par défaut).
_parent Ouvre le document lié dans le cadre parent.
_Haut Ouvre le document lié dans le corps de la fenêtre.

Les paramètres valides sont:

Paramètres La description
'hauteur = [ px ]' La hauteur de la fenêtre, la valeur minimale étant 100.
'largeur = [ px ]' La largeur de la fenêtre, la valeur minimale étant 100.
'left = [ px ]' La position gauche de la fenêtre.
'top = [ px ]' La position la plus haute de la fenêtre (IE uniquement).
'location = [ booléen ]' Affichage ou non du champ d'adresse (la valeur par défaut est 1).
'menubar = [ boolean ]' Affichage ou non de la barre de menus (la valeur par défaut est 1).
'redimensionnable = [ booléen ]' Indique si la fenêtre est redimensionnable (la valeur par défaut est 1).
'barres de défilement = [ booléen ]' Affichage ou non des barres de défilement (la valeur par défaut est 1).
'statut = [ booléen ]' S'il faut ou non ajouter une barre d'état (la valeur par défaut est 1).
'titlebar = [ booléen ]' Afficher ou non la barre de titre. Ceci est ignoré sauf si l'application appelante est une application HTML ou une boîte de dialogue sécurisée (la valeur par défaut est 1);
'toolbar = [ boolean ]' Affichage ou non de la barre d'outils du navigateur (la valeur par défaut est oui).

Il convient également de noter que si l'argument d'URL que vous indiquez est le nom d'un script (c'est-à-dire qu'il ne commence pas par "http: //") ET que le script s'appelle gmcallback_* alors vous pouvez obtenir des flux directs dans un événement html5. GameMaker Studio 2 noms de fonction de GameMaker Studio 2 ne sont pas obscurcis s’ils commencent par gmcallback_.


Exemple:

home_but = clickable_add(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0");

Le code ci-dessus crée une icône DOM cliquable à la position (32, 32) de la page sur laquelle le canevas du jeu est exécuté. L'icône utilise l'image-objet référencée dans la page de texture en tant que "spr_MS_Home" et, lorsque vous cliquez dessus, l'icône ouvrira une nouvelle fenêtre pour l'URL spécifiée et avec les propriétés définies pour cette fenêtre.
