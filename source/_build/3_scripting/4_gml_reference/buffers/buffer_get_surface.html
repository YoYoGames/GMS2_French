buffer_get_surface

Syntaxe:

buffer_get_surface(buffer, surface, mode, offset, modulo);


Argument La description
buffer L'index du tampon à utiliser.
surface L'indice de la surface à utiliser.
mode Les données à extraire de la surface (voir la liste des constantes ci-dessous).
offset La valeur de décalage de données.
modulo Le décalage de la fin de la ligne au début de la prochaine


Retours: N / A


La description

Avec cette fonction, vous pouvez écrire les informations d’une surface sur un tampon donné. Le tampon doit avoir été créé précédemment et doit être un tampon de croissance aligné sur 1 octet, d'une taille approximative (en octets) de la surface que vous allez écrire. Vous pouvez choisir le "mode" de copie pour les données stockées, défini par les constantes listées ci-dessous, ainsi que les valeurs offset et modulo. Offset et modulo sont tous deux des valeurs d'octets, l'offset définissant la position de départ dans la mémoire tampon et modulo étant le nombre d'octets restants à la fin de chaque ligne (pour stocker des données supplémentaires). Notez que la fonction écrit chaque pixel de la surface dans la mémoire tampon à l'aide d'un formatage BGRA sur la cible Windows, mais sur d'autres cibles, il peut être différent selon le système d'exploitation ou même le périphérique.

Constant La description
buffer_surface_copy Copie toutes les données de la surface.


Exemple:

buffer_get_surface(buff, application_surface, buffer_surface_copy, 0, 0);

Le code ci-dessus copiera les données stockées dans la surface de l'application dans la mémoire tampon indexée dans la variable "buff" sans valeur offset ni modulo.
