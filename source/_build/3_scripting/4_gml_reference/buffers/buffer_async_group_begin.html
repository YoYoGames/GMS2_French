buffer_async_group_begin

Syntaxe:

buffer_async_group_begin(groupname);


Argument La description
groupname Le nom du groupe (en tant que chaîne).


Retours: N / A


La description

Cette fonction est appelée lorsque vous souhaitez commencer l'enregistrement de plusieurs tampons dans plusieurs fichiers. Le "nom de groupe" est une chaîne et sera utilisé comme le nom du répertoire où les fichiers seront sauvegardés, et devrait être utilisé comme partie du chemin du fichier lors du chargement des fichiers dans l'EDI plus tard (en utilisant l'un des buffer_load les fonctions). Cette fonction est uniquement destinée à être utilisée avec buffer_save_async() fonction et vous devez également terminer la définition de sauvegarde en appelant buffer_async_group_end() fonction sinon les fichiers ne seront pas sauvegardés.

Notez que pour les plates-formes de console (comme PS4 par exemple), le "nom de groupe" sera utilisé comme description de l'emplacement de sauvegarde, et l'utilisation de cette fonction peut vous éviter d'afficher l'interface utilisateur pour chaque fichier sauvegardé.


Exemple:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

Le code ci-dessus démarre un groupe de tampons puis le définit pour enregistrer 4 fichiers de manière asynchrone. La définition du groupe est alors terminée (à quel moment la sauvegarde commencera).
