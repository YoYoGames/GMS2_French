buffer_async_group_begin

Syntaxe:

buffer_async_group_begin(groupname);


Argument La description
groupname Le nom du groupe (sous forme de chaîne).


Retours: N / A


La description

Cette fonction est appelée lorsque vous souhaitez commencer l’enregistrement de plusieurs mémoires tampons dans plusieurs fichiers. Le " groupname " est une chaîne qui sera utilisée comme nom de répertoire dans lequel les fichiers seront enregistrés. Elle doit être utilisée dans le chemin du fichier lors du chargement ultérieur des fichiers dans l'EDI (à l'aide de l'un des fichiers suivants). buffer_load les fonctions). Cette fonction ne doit être utilisée qu'avec le buffer_save_async() fonction et vous devez également terminer la définition de sauvegarde en appelant buffer_async_group_end() sinon, les fichiers ne seront pas sauvegardés.

Notez que pour les plates-formes de console (comme la PS4 par exemple), le "nom du groupe" sera utilisé comme description de l'emplacement de sauvegarde, et l'utilisation de cette fonction peut vous aider à éviter l'affichage de l'interface utilisateur pour chaque fichier en cours d'enregistrement.


Exemple:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

Le code ci-dessus démarre un groupe de tampons, puis le configure pour enregistrer 4 fichiers de manière asynchrone. La définition du groupe est alors terminée (à quel point la sauvegarde commencera).
