Fonctions de la couche Tilemap

L' GameMaker Studio 2 pièce GameMaker Studio 2 vous permet d'ajouter des mosaïques dans n'importe quelle pièce à l'aide de calques. Comme son nom l'indique, une couche de tilemap est simplement une collection de mosaïques de mosaïques réputées être à la même profondeur dans la pièce. En superposant des couches, vous pouvez créer des éléments superposés ou superposés. Vous pouvez également contrôler certains aspects des calques à partir de code, en ajoutant ou supprimant des éléments, ou en modifiant certaines propriétés du calque ou son contenu au moment de l'exécution. Notez que dans l'éditeur de salle, vous êtes limité à un tilemap par couche (le calque de mosaïque ), mais lorsque vous les utilisez dans le code, vous pouvez affecter plusieurs tilemaps à un seul calque, chaque carte ayant son propre identifiant et propriétés uniques. alors:

  • Le jeu de tuiles est la ressource que vous utilisez pour créer des couches de tuiles et des tilemaps
  • La couche de mosaïque correspond à ce que vous créez dans l’éditeur de salle pour y ajouter des mosaïques.
  • Le tilemap est ce que vous appelez la collection de carreaux qui sont ajoutés à un calque, soit dans l'éditeur de salle, soit via le code, en tant qu'élément unique sur ce calque.

Fonctions de tuile

Outre les fonctions répertoriées ci-dessus pour définir des tilemaps, vous disposez également d'un certain nombre de fonctions permettant de manipuler les données de différentes tuiles. Un "blob" de données de mosaïque est constitué de l'index de mosaïque et de quelques bits supplémentaires pour indiquer si la mosaïque a été tournée, retournée ou mise en miroir. Vous avez également la possibilité d'ajouter votre propre masque de bits aux données de mosaïque, bien que cela ne soit destiné qu'aux utilisateurs expérimentés, car vous masquerez les bits réservés à l'index de mosaïque. En gros, 19 bits sont réservés pour vos index de mosaïque. vous n'en utilisez que 8, il vous reste alors onze bits que vous pouvez utiliser pour créer un masque personnalisé. Ce masque peut ensuite être utilisé et vérifié dans le code pour créer des collisions de mosaïques personnalisées ou autres. Cependant, vous n’auriez normalement pas besoin de créer vos propres blobs de tiledata et vous utiliseriez plutôt la fonction tilemap_get pour obtenir le tiledata, vous le manipuleriez à l’aide des fonctions ci-dessous, puis vous le paramètreriez de nouveau avec tilemap_set.
