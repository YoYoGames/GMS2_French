tilemap_set_global_mask

La description

Cette fonction peut être utilisée pour définir le masque de bit de tuile pour tous les tilemaps du jeu. L'idée est que, si vous avez, par exemple, de très petits ensembles de pavés de 16 x 16 pavés auront un total de 256 pavés différents dans chaque ensemble de pavés, ce qui n'utilise que 8 bits de votre index dans le "blob" de données de pavé (de un 19 bits de données disponibles). Ces bits peuvent être "masqués", puis le reste des données de pavé peut être utilisé pour stocker des valeurs supplémentaires.

Un moyen simple d’obtenir la valeur initiale du masque d’index est de prendre le nombre de carreaux dans l’ensemble de tuiles et de soustraire 1; ainsi, un ensemble de carreaux 16x16 comporte 256 carreaux et une valeur de masque de 255 (ou $ff en hexadécimal). Le jeu de tuiles comporte un nombre de tuiles qui ne correspond pas à une puissance de deux, puis arrondissez à la puissance de deux la plus proche pour obtenir le masque. Par exemple, un jeu de tuiles au format 20x20 contient 400 tuiles, que vous arrondiriez à 512, et obtenez donc une valeur d'index de masque de 511 ( $1ff en hexadécimal). En plus du masque d'index, vous souhaiterez normalement conserver les valeurs d'inversion, de rotation et de miroir affectées à une mosaïque. Par conséquent, quelques constantes peuvent être utilisées. Celles-ci peuvent être "ou" associées à la valeur du masque d'index pour conserver ces bits de données. Les constantes suivantes peuvent être utilisées (voir l'exemple de code ci-dessous pour plus d'informations):
Constant La description
tile_rotate Utilisé pour définir / obtenir le bit de rotation d'un blob de données de mosaïque.
tile_mirror Utilisé pour définir / obtenir le bit miroir d'un blob de données de mosaïque.
tile_flip Utilisé pour définir / obtenir le bit de retournement d'un blob de données de mosaïque.
tile_index_mask Constante spéciale destinée à "et" avec le blob de données de mosaïque pour extraire l'index de mosaïque.


Le masquage de bit Tilemap est une fonctionnalité avancée qui vous permet d’utiliser des bits du "blob" de données de mosaïque pour votre propre usage. Par exemple, l'utilisateur pourrait réserver un bit pour signifier "solide" dans la structure de mosaïques de données, puis vérifier la présence de collisions rapides. Fondamentalement, le masque de bits est "et" -ed par rapport aux données de mosaïque lorsque le tilemap est dessiné (de sorte que cela n'affecte aucune des autres fonctionnalités), ce qui signifie que les bits qui ne sont pas dans le masque sont ignorés, mais peuvent toujours être lu et écrit par l'utilisateur pour ses propres besoins. Notez qu'il existe deux masques différents, l'un global et l'autre par carte, et qu'ils sont "et" assemblés en interne par GameMaker Studio 2 avant d'être utilisés pour le dessin.


Syntaxe:

tilemap_set_global_mask(mask)


Arguments:

Argument La description
mask La valeur de masque à utiliser


Résultats:

N/A


Exemple:

var mask = tile_mirror | tile_flip | tile_rotate | 255;
tilemap_set_global_mask(mask);

Le code ci-dessus définit le masque de bits de tilemap global sur 8 bits, en préservant les valeurs d'inversion, de symétrie et de rotation pour chaque mosaïque de la carte.
