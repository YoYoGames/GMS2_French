audio_create_buffer_sound


La description

Avec cette fonction, vous pouvez créer un nouveau son à partir du contenu d’un tampon. Le tampon aura été créé précédemment (voir les fonctions du tampon pour savoir comment procéder) et des données y ont été ajoutées ou chargées. Vous le transmettez ensuite à cette fonction avec le format de données (uniquement buffer_u8 ou buffer_s16 actuellement pris en charge), le taux d'échantillonnage (qui peut être compris entre 1000 Hz et 48 000 Hz) et un décalage dans la mémoire tampon à partir duquel les données sont extraites. Vous devez également fournir le nombre d'échantillons dans la mémoire tampon et les canaux requis par le son. Ces canaux sont définis par l'une des constantes suivantes:

Constant La description
audio_mono Audio mono (monocanal).
audio_stereo Audio stéréo (double canal).
audio_3D Audio 3D (5.1).


Notez qu'après avoir créé un son, vous devez libérer l'index de pointeur qui lui est associé lorsqu'il n'est plus nécessaire d'utiliser la fonction. audio_free_buffer_sound(). Si vous ne le faites pas, puis réaffectez la variable ou changez de pièce, etc., l'identifiant du son sera perdu et vous aurez une fuite de mémoire. Notez également que vous ne pouvez pas supprimer le tampon si aucun son n’a été créé et si le son n’a pas été libéré au préalable. Donc, vous devez d'abord libérer le son (ou les sons), puis supprimer le tampon. Il est également intéressant de noter que l'ajout de quelque chose dans la mémoire tampon ou la modification de la taille de la mémoire tampon après la création d'un son donnera des résultats inattendus et qu'il n'est pas recommandé. Une fois que vous avez commencé à créer des sons à partir d'une mémoire tampon, vous ne devez pas les manipuler d'une autre manière par la suite.

IMPORTANT! Cette fonction n'est pas disponible avec la licence d'évaluation du produit.


Syntaxe:

audio_create_buffer_sound(bufferId, bufferFormat, bufferRate, bufferOffset, bufferLength, bufferChannels);


Argument La description
bufferId L'ID du tampon à utiliser.
bufferFormat Le format des données dans le tampon ( buffer_u8 ou buffer_s16 ).
bufferRate Le taux d'échantillonnage des données dans la mémoire tampon.
bufferOffset Le décalage dans la mémoire tampon pour lire les données de l'échantillon (en octets).
bufferLength La longueur de la mémoire tampon (le nombre d'octets de données audio, sans l'en-tête).
bufferChannels Les canaux à utiliser parmi l’une des constantes répertoriées ci-dessous.


Résultats:

Index


Exemple:

var rate = 44100;
var hertz = irandom_range(220, 880);
var samples = 44100;
var bufferId = buffer_create(rate, buffer_fast, 1);
var buffer_seek(bufferId, buffer_seek_start, 0);
var num_to_write = rate / hertz;
var length = buffer_get_size(bufferId) ;
var val_to_write = 1;
for (var i = 0; i < (samples / num_to_write) + 1; i++;)
   {
   for (var j = 0; j < num_to_write; j++;)
      {
      buffer_write(bufferId, buffer_u8, val_to_write * 255);
      }
   val_to_write = (1 - val_to_write);
   }
soundId = audio_create_buffer_sound(bufferId, buffer_u8, rate, 0, length, audio_stereo);

Ce qui précède crée un tampon puis le remplit de manière procédurale avec des données. Ces données sont ensuite utilisées pour créer un nouveau son, qui est stocké dans la variable "soundId".
