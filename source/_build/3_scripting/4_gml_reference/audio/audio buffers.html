Tampons audio


Vous pouvez utiliser des tampons audio pour charger, sauvegarder et manipuler des fichiers audio dans vos jeux. Ces tampons sont préalablement créés à l'aide des fonctions de tampon appropriées. Vous pouvez ensuite utiliser les fonctions suivantes pour créer un son à utiliser dans votre projet:

  1. audio_create_buffer_sound
  2. audio_free_buffer_sound

Notez qu'une fois que vous avez créé un son à partir d'une mémoire tampon audio, vous ne créez pas un nouveau son en mémoire, mais vous pointez plutôt sur la position dans la mémoire tampon où le son est stocké. Vous ne devez pas modifier ce tampon après qu'un son lui a été associé, sinon vous obtiendrez des résultats inattendus et vous ne pourrez pas supprimer le tampon tant qu'un son lui est associé. Au lieu de cela, commencez par libérer le son (ou les sons), puis supprimez le tampon. Une fois que vous avez créé votre nouvelle ressource audio à partir d’un tampon, vous pouvez utiliser sa valeur d’index dans l’une des fonctions audio habituelles pour la lire, en modifier le gain ou la hauteur, la mettre en pause, etc. Notez que le tampon utilisé pour charger le L’audio ne supporte que les sons PCM, les formats tels que OGG ne seront pas supportés.

Une autre caractéristique de l'audio en mémoire tampon est que vous pouvez utiliser une mémoire tampon pour créer une file d'attente audio qui peut ensuite être diffusée en streaming sur le périphérique exécutant votre projet. Une file d'attente audio utilise un tampon régulier que vous avez précédemment rempli de données audio et vous permet de "pointer" vers certaines de ses parties et d'indiquer à GameMaker Studio 2: Studio de diffuser l'audio dans un ordre donné à partir de ce tampon. Les fonctions disponibles pour les files d'attente audio sont les suivantes:

  1. audio_create_play_queue
  2. audio_free_play_queue
  3. audio_queue_sound

Une fois la file créée, vous pouvez utiliser l’indice renvoyé comme vous le feriez pour tout son normal pour le lire, le suspendre, le redémarrer ou modifier son gain et sa hauteur. La seule chose que vous ne pouvez pas faire est d'obtenir la longueur de la piste ou de définir la position de la piste. Sinon, une file d'attente audio peut être utilisée comme n'importe quel autre son. Notez également que certaines fonctions de la file d'attente audio déclenchent également un événement de lecture audio asynchrone.

Les tampons audio vous permettront également d'enregistrer du son à partir de n'importe laquelle des sources disponibles (comme les microphones ou le système d'exploitation), qui peuvent ensuite être mises en file d'attente et diffusées à l'aide des fonctions ci-dessus (par exemple). L'enregistrement audio déclenchera un événement d'enregistrement audio asynchrone qui contiendra des informations spécifiques sur ce qui est en cours d'enregistrement.

Pour l'enregistrement audio, les fonctions suivantes sont disponibles:

  1. audio_start_recording
  2. audio_stop_recording
  3. audio_get_recorder_count
  4. audio_get_recorder_info
REMARQUE: la plupart des plates-formes, à l' exception de HTML5, prennent en charge l'enregistrement audio sous quelque forme que ce soit, mais cela ne signifie pas que tous les périphériques le permettent, même si la plate-forme le permet. Vous devez donc toujours vérifier que le audio_get_recorder_count() function renvoie une valeur supérieure à 0 pour vérifier que les dispositifs d’enregistrement sont disponibles avant d’utiliser les autres fonctions d’enregistrement.
