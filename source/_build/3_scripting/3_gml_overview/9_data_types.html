Types de données


Dans les sections précédentes, nous avons traité des variables et de leurs règles de portée, mais peu de choses ont été dites sur les différents types de données qu'une variable peut stocker. C'est pourquoi ce chapitre explique les différents types et leur utilité.

Avant de continuer, expliquons brièvement ce que nous entendons par "types de données". Lorsque vous créez une variable, elle peut être utilisée pour conserver des informations et lorsque vous appelez une fonction GML, elle peut également renvoyer des informations. Toutefois, ces informations peuvent prendre différentes "formes", comme par exemple un nombre réel ou une chaîne. Ces différents types de valeurs utilisées sont appelés types de données. Lorsque vous utilisez GML, ils peuvent être l'un des suivants:

Une chaîne est simplement n'importe quel texte placé entre guillemets "...". Vous pouvez effectuer certaines opérations sur les chaînes, comme ajouter deux chaînes ensemble pour créer une chaîne plus longue (concaténation). Vous pouvez également modifier les propriétés de la chaîne et même en extraire des nombres réels. Pour plus d'informations sur les chaînes et les fonctions de chaîne, voir: Référence du langage GameMaker - Chaînes.


Les nombres réels sont toute valeur qui n'est pas une chaîne, ni aucun des autres types de données possibles. Ainsi, 124, 45639.566546456, 0, -45.5 etc... sont tous des exemples de nombres réels. Tous les nombres réels sont stockés sous forme de valeurs à virgule flottante double précision (même des valeurs apparemment entières). Vous pouvez donc rencontrer de légères erreurs d'arrondi lorsque vous les utilisez. Pour plus d'informations sur cette fonction et sur d'autres fonctions liées aux nombres, veuillez consulter: Référence GML - Nombres réels.

Notez que, bien que les variables créées dans GameMaker Studio 2 soient toutes stockées sous forme de nombres à virgule flottante double précision, vous pouvez toujours utiliser d'autres formats pour les extensions. Ceux-ci peuvent être passés à GameMaker Studio 2 partir d’une extension, puis vérifiés à l’aide du is_ fonction, listée au bas de cette page.


Un tableau est un type de données spécial qui peut être utilisé pour contenir plusieurs valeurs. Vous affectez le tableau à une variable, puis "remplissez" différents indices du tableau avec des valeurs. Ce tableau peut ensuite être transmis aux scripts et aux fonctions, référence par référence. Toutefois, si vous modifiez le tableau transmis, il deviendra alors une copie du tableau d'origine et devra donc être renvoyé à la variable d'origine. pour que le tableau soit mis à jour. Pour plus d'informations sur les tableaux, veuillez consulter: Présentation de GML - Tableaux.


GameMaker Studio 2 acceptera également les littéraux hexadécimaux comme valeurs légitimes. Les valeurs hexadécimales sont particulièrement courantes lorsque vous travaillez avec des couleurs, mais peuvent être utilisées partout où une valeur entière positive est requise. Notez que les valeurs hexadécimales peuvent être formatées de l'une des deux manières suivantes:

$abcd
0xabcd

Par exemple, les valeurs suivantes peuvent être exprimées en hexadécimal, comme indiqué:

11406 = $2c8e = 0x2c8e
16777215 = $ffffff = 0xffffff


Un booléen est simplement une valeur qui peut être soit true ou false. Notez qu'actuellement GameMaker Studio 2 interprétera un nombre réel égal ou inférieur à 0,5 comme un false valeur, et tout nombre réel supérieur à 0,5 comme étant true. Cela ne signifie cependant pas que vous devriez vérifier 1 et 0 (ou tout autre nombre réel) pour vrai et faux, comme vous sont également fournis avec les constantes true et false ce qui devrait toujours être utilisé dans votre code pour éviter tout problème si de vrais types de données booléens étaient ajoutés dans une future mise à jour.

Vous pouvez convertir n'importe quel nombre réel en une valeur implicitement booléenne à l'aide de la fonction suivante:

  • bool

Un pointeur est un type de données qui "pointe" sur un emplacement de mémoire. Vous ne pouvez pas effectuer d'opérations sur un pointeur. Ce dernier n'est utilisé que pour certaines fonctions très spécifiques, telles que l'obtention d'une texture ou d'une adresse tampon de la mémoire pour une autre fonction. Pour des exemples de fonctions qui renvoient un pointeur, vous pouvez voir buffer_get_address ou sprite_get_texture.

Il existe également une fonction permettant de vérifier si une valeur est un pointeur (voir " Vérification des types de données " ci-dessous) et une fonction permettant de convertir une valeur en pointeur:

  • ptr

Vous pouvez également utiliser (et recevoir) les constantes intégrées suivantes lorsque vous utilisez des pointeurs:

Constant La description
pointer_null Cette constante indique que le pointeur ne pointe pas vers quelque chose de significatif (le même que NULL en C ++ ou null en C #)
pointer_invalid Cette constante signifie simplement que la valeur n'est pas un pointeur valide

Une énumération est un "énumérateur" et vous permet essentiellement de créer votre propre type de données limité avec une liste de valeurs constantes. Les énumérations sont des variables de portée globale (ceci est implicite et ne nécessite pas de " global "préfixe) et ils ont la structure suivante:

enum <variable>{<constant> [= <value>]}

Dans l'exemple suivant, nous créons une énumération pour les couleurs de l'arc-en-ciel et lui affectons diverses constantes et valeurs par défaut:

enum rainbowcolors {
   red,
   orange,
   yellow,
   green,
   blue,
   indigo,
   violet
   }

Les entrées enum ne peuvent être que des nombres entiers ou des expressions avec des énumérations précédentes évaluées en un nombre entier, et sont numérotées par défaut à partir de 0, ainsi notre exemple donné ci-dessus serait par défaut red = 0, orange = 1, yellow = 2, etc...

Vous pouvez également affecter des valeurs aux variables enum au moment de la création:

enum rainbowcolors {
   red = 5,
   orange = 5 * 2,
   yellow = 15,
   green = 20,
   blue = 25,
   indigo = 30,
   violet = 35 * enum_test.entry
   }

Notez que dans l'exemple ci-dessus, nous utilisons une autre énumération pour créer une expression pour "violet". Cela ne fonctionne que si l'énum référencé a été créé avant l'enum qui l'utilise dans une expression, mais cela ne fonctionnera pas pour les variables ou les fonctions, car la valeur enum doit pouvoir être évaluée en tant que constante lors de la compilation. Notez également que toutes les valeurs enum sont évaluées en valeurs entières. Lorsque vous créez la vôtre, vous devez savoir que seules les valeurs entières sont autorisées pour que les enums fonctionnent. Cette valeur peut être n'importe quel nombre entier qu'un nombre à double précision à virgule flottante peut représenter.

Pour accéder ultérieurement à la valeur dans un type d’énum donné, vous pouvez utiliser le point "." méthode, comme ceci:

var value = rainbowcolours.green;

Notez que vous ne pouvez pas modifier les valeurs d'une constante enum après sa création.


Une valeur indéfinie (également appelée "valeur" nulle "") est une expression dans laquelle une expression n'a pas de valeur correcte, bien qu'elle soit syntaxiquement correcte, et doit donc renvoyer quelque chose. Par exemple, supposons que ds_map et utiliser la fonction ds_map_find_value(). Maintenant, que se passe-t-il lorsque la carte n'a pas la valeur recherchée? Eh bien, puisque la fonction est correctement formatée et que le problème est qu’il n’existe pas de telle valeur, elle renverrait la constante. undefined, et vous pouvez vérifier cette constante comme vous le feriez pour true ou toute autre valeur.


Vérification des types de données

GameMaker Studio 2 vous permet de vérifier le type de données d'une variable donnée à l'aide des fonctions suivantes:

  • is_string
  • est vrai
  • is_bool
  • is_array
  • is_ptr
  • is_int32
  • is_int64
  • is_vec3
  • is_vec4
  • is_matrix
  • is_undefined
  • Type de
