Types de données


Dans les sections précédentes, nous avons couvert les variables et leurs règles de portée, mais peu a été dit sur les différents types de données qu'une variable peut stocker. Par conséquent, ce chapitre explique les différents types et à quoi ils peuvent être utilisés.

Avant de continuer, expliquons brièvement ce que nous entendons par "types de données". Lorsque vous créez une variable, elle peut être utilisée pour stocker des informations et lorsque vous appelez une fonction GML, elle peut également renvoyer des informations. Cependant, cette information peut venir de diverses "saveurs", comme elle peut (par exemple) être un nombre réel ou une chaîne de caractères. Ces différents types de valeurs utilisés sont appelés types de données et lors de l'utilisation de GML, ils peuvent être l'un des suivants:

Une chaîne est simplement un texte qui a été placé entre guillemets ('...' ou '...'). Vous pouvez effectuer certaines opérations sur des chaînes, comme ajouter deux chaînes ensemble pour créer une chaîne plus longue (concaténation) et vous pouvez également modifier les propriétés de la chaîne et même en extraire des nombres réels. Pour plus d'informations sur les fonctions de chaîne, consultez: Référence du langage GameMaker - Chaînes.


Les nombres réels sont toutes les valeurs qui ne sont pas une chaîne, ni aucun des autres types de données possibles. Donc, 124, 45639.566546456, 0, -45.5 etc... sont tous des exemples de nombres réels. Tous les nombres réels sont stockés sous forme de valeurs à virgule flottante 32 bits (valeurs entières paires), de sorte que vous pouvez rencontrer de légères erreurs d'arrondi lorsque vous les traitez. Pour plus d'informations sur cette fonction et d'autres fonctions liées aux nombres, veuillez consulter: Référence GML - Nombres réels.

Notez que les variables créées dans GameMaker Studio 2 sont toutes stockées sous la forme de nombres à virgule flottante double précision, mais vous pouvez utiliser d'autres formats pour les extensions. Ceux-ci peuvent être transmis dans GameMaker Studio 2 partir d'une extension, puis vérifiés en utilisant les is_ fonction, répertoriée en bas de cette page.


Un tableau est un type de données spécial qui peut être utilisé pour contenir plusieurs valeurs. Vous affectez le tableau à une variable, puis "remplissez" différents index du tableau avec des valeurs. Ce tableau peut ensuite être transmis aux scripts et aux fonctions par référence, mais si vous modifiez le tableau passé, il deviendra une copie du tableau original et devra donc être retourné à la variable d'origine pour que le tableau soit mis à jour. Pour plus d'informations sur les tableaux, veuillez consulter: Présentation GML - Tableaux.


Un booléen est simplement une valeur qui peut être soit true ou false. Notez que GameMaker Studio 2 ne prend pas en charge les valeurs booléennes "true" et accepte tout nombre réel inférieur à 0.5 comme valeur false, et tout nombre réel égal à (ou supérieur à) 0.5 comme vrai. Cela ne signifie pas cependant que vous devriez vérifier 1 et 0 (ou tout autre nombre réel) pour vrai et faux, car vous êtes également fourni avec les constantes true et false qui devrait toujours être utilisé dans votre code pour éviter tout problème si de vrais types de données booléens devaient être ajoutés dans une future mise à jour.


Un pointeur est un type de données qui "pointe" vers un emplacement de mémoire. Vous ne pouvez pas effectuer d'opérations sur un pointeur et il est utilisé uniquement pour certaines fonctions très spécifiques, comme l'obtention d'une texture ou d'une adresse de mémoire tampon pour une autre fonction. Pour des exemples de fonctions qui renvoient un pointeur, vous pouvez voir buffer_get_address ou sprite_get_texture.

Il y a aussi une fonction pour vérifier si une valeur est un pointeur (voir " Vérification des types de données " ci-dessous) et une fonction pour convertir une valeur en pointeur:

  • ptr

Vous pouvez également utiliser (et obtenir retourné) les constantes intégrées suivantes lors de l'utilisation de pointeurs:

Constant La description
pointer_null Cette constante indique que le pointeur ne pointe sur rien de significatif (identique à NULL en C ++ ou null en C #)
pointer_invalid Cette constante signifie simplement que la valeur n'est pas un pointeur valide

Un enum est un "énumérateur", et il vous permet essentiellement de créer votre propre type de données limité avec une liste de valeurs constantes. Les énumérations sont des variables de portée globales (implicites et non requises) global "suffixe" et ils ont la structure suivante:

enum <variable>{<constant> [= <value>]}

Dans l'exemple suivant, nous créons une énumération pour les couleurs de l'arc-en-ciel et lui assignons différentes constantes et valeurs par défaut:

enum rainbowcolors {
   red,
   orange,
   yellow,
   green,
   blue,
   indigo,
   violet
   }

Les entrées enum ne peuvent être que des nombres réels ou des expressions avec des énumérations précédentes, et par défaut sont numérotées de 0 vers le haut, donc notre exemple donné ci-dessus serait par défaut red = 0, orange = 1, yellow = 2, etc... Vous pouvez également assigner des valeurs aux variables enum au moment de la création:

enum rainbowcolors {
   red = 5,
   orange = 5 * 2,
   yellow = 15,
   green = 20,
   blue = 25,
   indigo = 30,
   violet = 35 * enum_test.entry
   }

Remarquez dans l'exemple ci-dessus que nous utilisons une autre énumération pour créer une expression pour "violet". Cela ne fonctionne que si l'énumération référencée a été créée avant l'énumération qui l'utilise dans une expression, mais cela ne fonctionnera pas pour les variables ou les fonctions, puisque la valeur enum doit pouvoir être évaluée comme une constante lors de la compilation. Notez également que toutes les valeurs enum sont évaluées en valeurs entières et lorsque vous créez la vôtre, vous devez savoir que seules les valeurs entières sont autorisées pour que les énumérations fonctionnent.

Pour accéder ultérieurement à la valeur dans un type enum donné, vous pouvez utiliser le point "." méthode, comme ceci:

var value = rainbowcolours.green;

Notez que vous ne pouvez pas modifier les valeurs d'une constante enum après sa création.


Une valeur indéfinie (également appelée "valeur nulle") est une valeur où une expression n'a pas de valeur correcte, bien qu'elle soit syntaxiquement correcte, et qu'elle doive donc renvoyer quelque chose. ds_map et utilisez la fonction ds_map_find_value(). Maintenant, que se passe-t-il lorsque la carte n'a pas la valeur recherchée? Eh bien, puisque la fonction est correctement formatée, et le problème est que la valeur n'existe pas, alors il retournera la constante undefined, et vous pouvez vérifier cette constante que vous vérifieriez pour true ou toute autre valeur.


Vérification des types de données

GameMaker Studio 2 vous permet de vérifier le type de données d'une variable donnée en utilisant les fonctions suivantes:

  • is_string
  • est vrai
  • is_bool
  • is_array
  • is_ptr
  • is_int32
  • is_int64
  • is_vec3
  • is_vec4
  • is_matrix
  • is_undefined
  • Type de
