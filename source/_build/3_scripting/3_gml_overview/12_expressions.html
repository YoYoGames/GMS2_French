Expressions


Une expression est une phrase mathématique pouvant contenir des nombres ordinaires, des variables, des chaînes de caractères ou des fonctions GML ainsi qu'un ou plusieurs opérateurs (tels que additionner, soustraire, multiplier, etc.). Les valeurs utilisées dans une expression peuvent être des nombres réels (par exemple, 3.4 * 6), des nombres hexadécimaux commençant par un signe $ (par exemple, $00FFAA | $88FFAA ), des chaînes entre guillemets (par exemple, "bonjour" + "monde") ou plus compliquées. expressions utilisant plusieurs opérateurs et valeurs.

Pour les expressions, les opérateurs suivants existent (par ordre de priorité):

= - Utilisé pour affecter une valeur à une variable. Notez que cela peut également être utilisé pour comparer des variables dans GameMaker Studio 2 et que cela peut se voir dans les exemples et les codes d'autres personnes. Cependant, il s'agit d'un héritage d'anciennes versions de GameMaker et vous devez utiliser les opérateurs == pour comparer et = pour affecter, comme indiqué dans ces exemples:

a = 12;
speed = 5;
val = (old_val + 5);


&&, ||, ^^ (et, ou et xor) - Combine des valeurs booléennes pour donner la valeur true ou false. Si l’un des exemples suivants résout le problème true alors le code serait lancé:

if (a == b && c == d) {do something...}// and
if (a == b || c ==d) {do something...}// or
if (a == b ^^ c == d) {do something...} // xor


<, <=, ==,! =,>,> = - Ce sont des comparaisons et ne peuvent donner qu'une true ou false result (où true peut également être interprété en tant que 1 et false en tant que 0). Exemples d'utilisation:

if (a < b) {do something...}
if (a != b) {do something...}


|, &, ^, <<, >>: Vous pouvez effectuer des opérations au niveau des bits avec ceux-ci, où | = au niveau du bit ou, & = au niveau du bit et, ^ = au niveau du bit xor, << = décaler à gauche, >> = décaler à droite. Exemples d'utilisation:

x = (x & $ffffffe0) + 32;
if (y ^ $1f) > 0 {do something...};


Vous pouvez trouver des informations supplémentaires sur l'utilisation des opérateurs au niveau du bit et sur ce qu'ils font dans la section: Opérateurs au niveau du bit.

+, -, *, / - Ajoutez, soustrayez, multipliez et divisez. Exemples d'utilisation:

c = a * b;
str = a + "world";


++, - - Ajouter ou soustraire un à une valeur. Il est à noter que le fait de placer cette valeur avant ou après la valeur à ajouter ou à soustraire aura des résultats légèrement différents. Par exemple:

  • ++ a va incrémenter la variable et renvoyer la valeur incrémentée.
  • a ++ incrémentera la variable mais renverra la valeur avant son incrémentation.
Par conséquent, si vous avez quelque chose comme ça:

var a = 1;
show_debug_message(string(a++));
show_debug_message(string(++a));


La sortie de débogage serait 1 et 3. Voici quelques exemples d'utilisation:

for (var i = 0; i < 10; i++;) {do something...};
if hit == true score--;


REMARQUE: Sur les plates-formes cibles du compilateur YoYo (celles marquées (YYC)), ces expressions sont évaluées de gauche à droite, tandis que sur toutes les autres plates-formes cibles, elles sont évaluées de droite à gauche, ce qui signifie que:

val = max(num, ++num, num++);


donnera des résultats différents selon la plate-forme.

div, mod (%) - Division et modulo, où div vous donne le montant qu'une valeur peut être divisée en produisant seulement un quotient entier, tandis que mod ne vous donne que le reste d'une division. Notez que vous ne pouvez que div ou mod en utilisant des valeurs entières. Exemples d'utilisation:

secs = time mod 60;
time_str = string(time div 60);


De plus, les opérateurs unaires suivants existent:
  • !: booléen "pas", donc !true == false
  • -: annule la valeur réelle ou entière suivante (non valide pour les chaînes ou les booléens)
  • ~: nie la valeur suivante au niveau du bit

En tant que valeurs dans toutes les expressions, vous pouvez utiliser des nombres, des variables ou des fonctions qui renvoient une valeur. Les sous-expressions peuvent également être placées entre crochets. Tous les opérateurs travaillent pour des valeurs réelles, mais les comparaisons fonctionnent également pour des chaînes et "+" concatène des chaînes.


Ordre des opérations

Lorsque vous effectuez plusieurs opérations dans une même expression, il est très important d'utiliser des crochets () pour séparer l'ordre des opérations, car différentes plates-formes peuvent les exécuter différemment, à moins que cela ne soit explicitement indiqué. Par exemple, considérons le code suivant:

a = b == c || d;

Les différents compilateurs cibles effectueront les opérations dans différents ordres car elles ne sont pas explicitement affichées, ce qui donne des résultats "impairs" auxquels vous ne vous attendez pas lorsque vous jouez. pour éviter cela, utilisez le () pour séparer les parties, comme ceci:

a = (b == c || d); //better
a = ((b == c) || d); //best

Voici quelques exemples finaux des différentes expressions:

{
x = 23;
colour = $FFAA00;
str = "hello world";
y += 5;
x *= y;
x = y << 2;
x = 23 * ((2 + 4) / sin(y));
str = "hello" + "world";
b = (x < 5) && !(x==2 || x==4);
}


Accesseurs
