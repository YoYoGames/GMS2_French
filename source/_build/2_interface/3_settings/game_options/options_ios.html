Options iOS


Cette section décrit les différentes options à votre disposition pour contrôler la compilation de vos projets de jeu iOS. Les différentes sections sont:

La première partie de la section Général couvre les paramètres de construction de votre jeu. Pour commencer, définissez les orientations sur lesquelles votre jeu peut être exécuté pour une expérience utilisateur optimale. Les options disponibles sont:

  • Portrait
  • Portrait retourné
  • Paysage
  • Paysage retourné

Vous pouvez ensuite accéder à la section Support de périphérique, avec les options suivantes disponibles pour définir le type de périphérique à cibler:

  • iPhone ou iPod Touch: iPhone 4 et supérieur ou iPod Touch
  • iPad: appareils iPad
  • Les deux: iPhone / iPod et iPad

La deuxième chose à faire est de renseigner les informations sur le produit concernant votre jeu, y compris son nom d'affichage, son identifiant de paquet et les informations de version. Notez que l' ID de l'ensemble doit être au bon format d'URL inverse pour que la version finale du jeu fonctionne correctement, par exemple: com.[Company].[GameName]. Vous devez également GameMaker Studio ici l’ identifiant d’équipe que vous souhaitez utiliser pour signer l’application finale GameMaker Studio par GameMaker Studio 2 pour vous. Si vous le définissez ici, l'ID de l'équipe que vous avez fourni dans les préférences iOS sera prioritaire.


Ici, vous pouvez modifier les détails suivants liés à l'affichage de votre jeu, avec les options suivantes disponibles:

  • Interpoler les couleurs entre les pixels: active l'interpolation, qui "lisse" les pixels. Pour les graphismes nets en pixels, il devrait être désactivé, mais si vous avez de beaux mélanges alpha et des graphiques à bords lissés, il est préférable de les laisser. La valeur par défaut est désactivée.

  • Half iPad 1 Textures: l'iPad 1 dispose de beaucoup moins de mémoire de texture que les appareils plus récents. Vous pouvez donc activer cette option pour réduire de moitié la taille des textures si vous souhaitez cibler cet appareil ou obtenir une compatibilité maximale.

  • Mise à l'échelle: vous pouvez choisir ici de redimensionner la zone de dessin en conservant les proportions au sein de l'écran du périphérique (en ajoutant un "remplissage" sur les bords pour l'adapter à l'écran) ou en sélectionnant un étirement de la zone de dessin.

Enfin, il est possible de définir la taille de la page de texture. La taille par défaut (et la plus compatible) est 2048x2048, mais vous pouvez choisir entre 256x256 et 4096x4096. Il existe également un bouton intitulé Aperçu qui générera les pages de texture pour cette plate-forme, puis ouvrira une fenêtre afin que vous puissiez voir à quoi elles ressemblent. Cela peut être très utile si vous souhaitez voir comment les pages de texture sont structurées et éviter d’avoir des pages de texture plus grandes (ou plus petites) que nécessaire.

REMARQUE: Sachez que plus la taille de la page de texture est grande, moins votre jeu sera compatible.

Cette section vous permet d’ajouter les différentes icônes requises par les différents appareils et les différentes parties de l’App Store iOS. Ces icônes doivent être créées au format 24 bits. .png images à la taille spécifiée pour chacune d’elles.

Il est à noter que GameMaker Studio 2 dispose d'un outil Project Image Generator qui peut être utilisé pour créer automatiquement toutes les icônes requises pour toutes les plates-formes cibles auxquelles votre jeu est compilé. Si vous utilisez cet outil, vous devez réviser les images créées pour vous assurer qu’elles correspondent à vos besoins.

REMARQUE: lors de l'importation d'un projet plus ancien, l'avertissement suivant peut s'afficher: Cela se produit car la version actuelle de la cible iOS a modifié le format des icônes requises depuis le chargement du projet. Il vous sera demandé de recréer toutes les icônes, car elles auront été remplacées par les GameMaker Studio 2 génériques de GameMaker Studio 2 (vous pouvez utiliser l'outil de Project Image Generator mentionné ci-dessus pour le faire rapidement).

Cette section vous permet d'ajouter des graphiques distincts sous forme d'écrans de démarrage (l'image qui sera affichée brièvement pendant le chargement de votre application) pour chacun des différents périphériques cibles iOS. Ces images doivent être créées sous forme d’images individuelles 24 bits *.png sans transparence et à la taille appropriée pour chacun des périphériques et orientations requis. Vous pouvez également définir un délai minimum d'affichage des écrans de démarrage dans le menu déroulant situé en bas (de 0 à 10 secondes).

Il est à noter que GameMaker Studio 2 dispose d'un outil Project Image Generator qui permet de créer automatiquement toutes les images requises pour les différentes plates-formes cibles pour lesquelles votre jeu est compilé. Si vous utilisez cet outil, vous devez réviser les images créées pour vous assurer qu’elles correspondent à vos besoins.


Cette section permet d'activer différentes fonctionnalités sociales dans votre jeu Android. Les options suivantes peuvent être choisies:

  • Facebook: Si vous avez besoin des fonctions Facebook, vous devez cocher cette case, puis fournir l' identifiant et le nom de l'application Facebook. Notez que toutes les fonctionnalités de Facebook sont accessibles via une extension. Lorsque vous cochez pour la première fois la case Utiliser Facebook, il vous sera demandé de télécharger et d'installer l'extension Facebook à partir du Marketplace.

  • Activer les notifications Push: pour que votre jeu utilise les notifications Push, elles doivent être activées ici au préalable, sinon elles ne fonctionneront pas. Les notifications locales sur Android ne nécessitent pas "l'ID de l'expéditeur GCM", mais vous devez l'ajouter si vous souhaitez créer des notifications à distance.
